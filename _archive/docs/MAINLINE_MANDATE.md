# Airivo 主线开发总纲

日期：2026-04-29

状态：最高约束

## 总纲

所有开发人员必须只围绕 Airivo = A股半自动交易决策与执行工作台 这一唯一产品主线开展开发，严格遵循 [CONTINUOUS_DEVELOPMENT_GOVERNANCE_GUIDE.md](/Users/mac/2026Qlin/docs/CONTINUOUS_DEVELOPMENT_GOVERNANCE_GUIDE.md:1) 与 [REPOSITORY_RETENTION_AND_REORG_PLAN.md](/Users/mac/2026Qlin/docs/REPOSITORY_RETENTION_AND_REORG_PLAN.md:1)，以单一主线、单一事实源、治理优先、收敛优先、删除优先于叠加为最高约束，持续完成主线收口、事实统一、目录归位、历史退役、门禁固化、发布稳定与审计闭环，未经明确批准不得扩张边界、增加平行入口、引入第二真相源或继续堆积临时脚本，直到系统成为可持续开发、可解释、可审计、可稳定发布的产品系统；凡是不直接提升主线闭环、不减少系统复杂度、不强化单一事实源、不提升发布与审计稳定性的改动，原则上不得进入生产主线。

## 执行解释

这不是愿景描述，而是主线开发最高裁决规则。

任何设计、编码、重构、迁移、运维、发布活动，只要与本总纲冲突，都应视为偏航。

## 必答四问

所有主线开发在开始前必须回答：

1. 这次改动是否直接服务 Airivo 主线闭环？
2. 这次改动是否强化了单一事实源，而不是制造第二口径？
3. 这次改动是否让系统更收敛，而不是更分叉？
4. 这次改动是否让发布、审计、回滚更稳定，而不是更依赖个人经验？

如果其中两项答不清楚，不应进入生产主线。
