### 诊断1：数据库检查

### 诊断2：评分逻辑检查
- **问题**：❌ 基础分太低（30分）
- **表现**：普通股票只能得35-40分，达不到50分最低阈值
- **修复**：
  ```python

  score = 30  # 基础分30分
  
  # 修复后

  ```

### 诊断3：评分标准检查
- **问题**：❌ 各维度评分条件太严格
- **修复**：全面放宽8个维度的评分标准
  ```python
  # 价格位置：从0.3/0.5/0.7放宽到0.4/0.6/0.8
  # 成交量：从2.0/1.5/1.2降低到1.8/1.3/1.1
  # 均线：只要有均线就给分
  # 动量：正动量就给分
  ```

### 诊断4：板块匹配检查
- **问题**：❌ 板块匹配太严格，90%股票被过滤
- **表现**：只有2个行业匹配成功（软件服务、半导体）
- **修复**：
  ```python
  # 如果板块匹配率<20%，使用全部股票
  if match_rate < 20:
      logger.info("板块匹配率太低，使用全部股票")
      hot_stocks = stocks.copy()
  ```

### 诊断5：信号类型检查（最关键！）
- **问题**：❌ **即使评分达标，signal_types判断太严格导致无信号**
- **表现**：评分50+分，但`signal_types`为空，不产生信号
- **修复**：
  ```python
  # 修复前（太严格）
  if price_position < 0.35 and volume_surge > 1.3:
      signal_types.append('底部放量')
  if normalized_score >= 50 and signal_types:  # 必须有signal_types
  
  # 修复后（宽松+兜底）
  if price_position < 0.6 and volume_surge > 1.1:  # 放宽条件
      signal_types.append('底部放量')
  if len(signal_types) == 0:  # 兜底机制
      signal_types.append('综合信号')
  if normalized_score >= 50:  # 移除signal_types硬性要求
  ```
