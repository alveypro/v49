### 问题2：回测系统与策略中心评分逻辑不一致 🔴

**位置**: 回测系统第2498-2597行

**问题描述**:
回测系统使用的评分阈值与策略中心不同，可能导致回测结果不准确。

**详细分析**:

#### 策略中心（Tab1）:
```python
# v4.0策略中心默认阈值：60分
score_threshold_v4 = st.slider("评分阈值", 50, 90, 60, ...)
```

#### 回测系统:
```python
# v4.0回测阈值：60-85分
def _backtest_with_real_evaluator_v4(...):
    if 60 <= final_score <= 85:  # 🚨 硬编码！
        # 记录信号
```

**问题**:
- ⚠️ 回测使用 60-85分区间
- ⚠️ 策略中心用户可以选择 50-90分
- ⚠️ 如果用户选择85分以上，回测结果不反映实际策略

**影响**:
- 严重性：🔴 **高**
- 回测准确性：❌ 降低
- 用户决策：❌ 误导

**建议修复**:
```python
# 将阈值参数化
def _backtest_with_real_evaluator_v4(self, df, sample_size, holding_days, 
                                      min_score=60, max_score=85):  # 添加参数
    # 使用传入的阈值
    if min_score <= final_score <= max_score:
        # 记录信号
```

### 问题3：数据库连接未正确关闭（资源泄漏）🔴

**位置**: 多处

**问题描述**:
大量代码在异常情况下未正确关闭数据库连接。

**示例1** - 第4545-4548行:
```python
if stocks_df.empty:
    st.error(f"❌ 未找到符合条件的股票...")
    st.info("💡 提示：请先到Tab1...")
    conn.close()  # ✅ 这里关闭了
else:
    # 长时间处理...
    conn.close()  # ✅ 这里也关闭了
# 但如果中间抛出异常呢？ ❌
```

**问题**:
- ❌ 如果在处理过程中抛出异常，连接不会关闭
- ❌ 长时间运行会导致连接耗尽
- ❌ 可能导致"database is locked"错误

**影响**:
- 严重性：🔴 **高**
- 稳定性：❌ 降低
- 资源管理：❌ 不当

**建议修复**:
```python
# 使用上下文管理器
try:
    with sqlite3.connect(PERMANENT_DB_PATH) as conn:
        # 所有数据库操作
        stocks_df = pd.read_sql_query(query, conn)
        # ...
except Exception as e:
    st.error(f"错误: {e}")
# 自动关闭连接，即使异常也会关闭
```

### 问题4：市值单位转换不一致 ⚠️

**位置**: 多处

**问题描述**:
系统中市值单位转换存在多种方式，容易混淆。

**不同的转换方式**:

1. **方式1** - 第4532-4533行:
```python
cap_min_wan = cap_min_v4 * 10000  # 亿 → 万元
cap_max_wan = cap_max_v4 * 10000
```

2. **方式2** - 第4552-4553行:
```python
actual_min_mv = stocks_df['circ_mv'].min() / 10000  # 万元 → 亿
actual_max_mv = stocks_df['circ_mv'].max() / 10000
```

3. **方式3** - 第4625行:
```python
'流通市值': f"{row['circ_mv']/10000:.1f}亿"  # 万元 → 亿
```

**问题**:
- ⚠️ 数据库存储单位：万元
- ⚠️ 用户输入单位：亿元
- ⚠️ 显示单位：亿元
- ⚠️ 多次转换容易出错

**建议修复**:
创建统一的单位转换函数：
```python
def wan_to_yi(value_wan):
    """万元转换为亿元"""
    return value_wan / 10000

def yi_to_wan(value_yi):
    """亿元转换为万元"""
    return value_yi * 10000

# 使用示例
cap_min_wan = yi_to_wan(cap_min_v4)
actual_min_mv_yi = wan_to_yi(stocks_df['circ_mv'].min())
```

### 问题5：v6.0评分器的协同加分逻辑过于复杂 ⚠️

**位置**: `comprehensive_stock_evaluator_v6_ultimate.py` 第583-685行

**问题描述**:
协同加分系统有8种组合类型，每种都有复杂的条件判断。

**代码复杂度**:
```python
def _calculate_synergy_v6_strict(self, dim_scores: Dict, ind: Dict, 
                                 pct_chg, volume) -> Dict:
    bonus = 0
    combo_types = []
    
    # 1. 板块总龙头（极难！）- 🔥最高15分
    if (dim_scores['板块热度'] >= 22 and      # 8个条件
        dim_scores['资金流向'] >= 28 and
        dim_scores['短期动量'] >= 13 and
        dim_scores['龙头属性'] >= 9 and
        dim_scores['相对强度'] >= 7 and
        dim_scores['技术突破'] >= 4 and
        pct_chg > 6 and
        volume > 0):
        bonus += 15
        combo_types.append('板块总龙头')
    
    # ... 还有7种组合
```

**问题**:
- ⚠️ 嵌套条件过多，难以理解
- ⚠️ 硬编码的阈值难以调整
- ⚠️ 维护成本高
- ⚠️ 测试困难

**影响**:
- 可维护性：❌ 差
- 可测试性：❌ 差
- 性能：⚠️ 中等

**建议修复**:
使用配置驱动的方式：
```python
# 定义协同组合配置
SYNERGY_COMBOS = {
    '板块总龙头': {
        'bonus': 15,
        'conditions': {
            '板块热度': 22,
            '资金流向': 28,
            '短期动量': 13,
            '龙头属性': 9,
            '相对强度': 7,
            '技术突破': 4,
        },
        'extra_checks': lambda pct_chg, volume: pct_chg > 6 and volume > 0
    },
    # 其他组合...
}

def _calculate_synergy_v6_strict(self, dim_scores, ind, pct_chg, volume):
    bonus = 0
    combo_types = []
    
    for combo_name, config in SYNERGY_COMBOS.items():
        if self._check_combo_conditions(dim_scores, config, pct_chg, volume):
            bonus += config['bonus']
            combo_types.append(combo_name)
    
    return {'bonus': bonus, 'types': combo_types}
```
