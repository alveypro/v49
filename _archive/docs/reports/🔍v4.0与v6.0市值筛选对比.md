## 🎯 为什么v6.0能找到更多股票？

### 原因1：没有市值下限

**v4.0**：
- 市值<100亿的股票：❌ 不扫描
- 如：30亿的小盘妖股 → 错过

**v6.0**：
- 市值<100亿的股票：✅ 扫描
- 如：30亿的小盘妖股 → 能扫描到

### 原因2：没有市值上限

**v4.0**：
- 市值>1500亿的股票：❌ 不扫描
- 如：3000亿的贵州茅台 → 错过

**v6.0**：
- 市值>1500亿的股票：✅ 扫描
- 如：3000亿的贵州茅台 → 能扫描到

### 原因3：真正的全市场扫描

**v4.0的"全市场扫描"**：
```python
# 名字叫"全市场扫描"，但实际上是"全市场符合条件的扫描"
use_all_stocks_v4 = True  # 勾选了
# 但SQL仍然有WHERE条件：
# WHERE sb.circ_mv >= 1000000 AND sb.circ_mv <= 15000000
# 所以只是"扫描100-1500亿范围内的所有股票"
```

**v6.0的"全市场扫描"**：
```python
# 真正的全市场扫描，没有任何市值限制
scan_all_stocks = True  # 默认勾选
# SQL没有WHERE条件：
# SELECT * FROM stock_basic
# 所以是"扫描所有A股"
```

## 💡 建议：统一v4.0的逻辑

### 当前问题

v4.0虽然有"全市场扫描"选项，但实际上仍然限制在100-1500亿范围内，这可能导致：

1. **错过小盘妖股**：市值30-50亿的连板妖股无法扫描到
2. **错过超大盘蓝筹**：市值2000亿+的行业龙头无法扫描到
3. **用户误解**：勾选了"全市场扫描"，以为扫描所有股票，实际上只扫描100-1500亿

### 建议修复

**方案A：完全对齐v6.0逻辑（推荐）**

```python
# 默认勾选"全市场扫描"
use_all_stocks_v4 = st.checkbox(
    "🌍 全市场扫描（推荐）",
    value=True,
    help="扫描所有A股，不限制市值范围"
)

# 高级选项（折叠）
with st.expander("⚙️ 高级筛选选项（可选）"):
    cap_min_v4 = st.number_input(
        "最小市值（亿元）", value=0,  # 默认0，表示不限制
        help="0表示不限制"
    )
    cap_max_v4 = st.number_input(
        "最大市值（亿元）", value=0,  # 默认0，表示不限制
        help="0表示不限制"
    )

# SQL查询
if use_all_stocks_v4 and cap_min_v4 == 0 and cap_max_v4 == 0:
    # 真正的全市场扫描
    query = """
        SELECT DISTINCT sb.ts_code, sb.name, sb.industry, sb.circ_mv
        FROM stock_basic sb
        ORDER BY RANDOM()
    """
    stocks_df = pd.read_sql_query(query, conn)
else:
    # 按市值筛选
    cap_min_wan = cap_min_v4 * 10000 if cap_min_v4 > 0 else 0
    cap_max_wan = cap_max_v4 * 10000 if cap_max_v4 > 0 else 999999999
    query = """
        SELECT DISTINCT sb.ts_code, sb.name, sb.industry, sb.circ_mv
        FROM stock_basic sb
        WHERE sb.circ_mv >= ?
        AND sb.circ_mv <= ?
        ORDER BY RANDOM()
    """
    stocks_df = pd.read_sql_query(query, conn, params=(cap_min_wan, cap_max_wan))
```

**方案B：保持当前逻辑，但明确说明**

## 🎊 总结

### 核心区别

| 项目 | v4.0 | v6.0 |
|------|------|------|
| 默认市值限制 | ✅ 有（100-1500亿） | ❌ 无 |
| 真正全市场扫描 | ❌ 否（限制在100-1500亿） | ✅ 是 |
| 扫描股票数量 | ~1576只 | ~3000-5000只 |

### 为什么v6.0找到更多股票？

1. **没有WHERE条件**：v6.0默认查询所有股票，不筛选市值
2. **覆盖所有市值范围**：从小盘妖股到超大盘蓝筹，全覆盖
3. **真正的全市场扫描**：不是"符合条件的全市场"，而是"全A股市场"

### 建议

如果希望v4.0也能扫描更多股票，建议：
1. 将默认市值改为0-0（表示不限制）
2. 对齐v6.0的逻辑，默认真正的全市场扫描
3. 高级选项中才设置市值范围
