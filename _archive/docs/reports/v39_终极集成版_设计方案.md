### 模块2：智能仓位管理系统（新增）

**作用**：根据市场环境、个股质量、风险水平动态调整仓位

```python
def calculate_position_size(
    market_env,      # 市场环境
    sector_stage,    # 板块生命周期
    leader_score,    # 龙头评分
    risk_level       # 风险水平
):
    基础仓位 = 20%
    
    # 1. 市场环境调整
    if market_env == '牛市':
        基础仓位 *= 1.5  # 30%
    elif market_env == '熊市':
        基础仓位 *= 0.5  # 10%
    
    # 2. 板块生命周期调整
    if sector_stage == '萌芽期':
        基础仓位 *= 1.2  # 最佳时机，加仓
    elif sector_stage == '爆发期':
        基础仓位 *= 0.3  # 过热，降低
    
    # 3. 龙头评分调整
    if leader_score >= 85:
        基础仓位 *= 1.3  # 龙头溢价
    elif leader_score < 70:
        基础仓位 *= 0.7  # 质量折扣
    
    # 4. 风险水平调整
    if risk_level == '低风险':
        基础仓位 *= 1.2
    elif risk_level == '高风险':
        基础仓位 *= 0.6
    
    # 限制在5-30%之间
    最终仓位 = max(5%, min(30%, 基础仓位))
    
    return 最终仓位
```

**实战案例：**
```
情况1：牛市 + 萌芽期 + 龙头股(85分) + 低风险
仓位 = 20% × 1.5 × 1.2 × 1.3 × 1.2 = 67% → 限制到30%

情况2：熊市 + 爆发期 + 跟随股(65分) + 高风险
仓位 = 20% × 0.5 × 0.3 × 0.7 × 0.6 = 1.3% → 提升到5%
```

### 模块3：多维风险控制矩阵（新增）

**作用**：从6个维度评估风险，避免踩雷

```python
风险维度：
1. 市场风险（大盘下跌风险）
2. 板块风险（板块过热风险）
3. 个股风险（ST、退市风险）
4. 流动性风险（成交量不足）
5. 估值风险（PE/PB过高）
6. 相关性风险（持仓板块高度相关）

风险评分（0-100）：
- 0-30：低风险 ✅
- 31-60：中等风险 ⚠️
- 61-100：高风险 ❌
```

**风险计算：**
```python
def calculate_risk_score(stock, market, portfolio):
    风险分数 = 0
    
    # 1. 市场风险（20分）
    if market.trend == '下跌':
        风险分数 += 20
    elif market.volatility > 3%:
        风险分数 += 15
    
    # 2. 板块风险（20分）
    if stock.sector_stage == '爆发期':
        风险分数 += 20  # 过热风险
    elif stock.sector_stage == '衰退期':
        风险分数 += 20  # 下跌风险
    
    # 3. 个股风险（20分）
    if 'ST' in stock.name:
        风险分数 += 20
    if stock.pe > 100 or stock.pe < 0:
        风险分数 += 15  # 估值异常
    
    # 4. 流动性风险（20分）
    if stock.avg_volume < 1000万:
        风险分数 += 20
    elif stock.turnover_rate < 1%:
        风险分数 += 15
    
    # 5. 估值风险（10分）
    if stock.pb > 10:
        风险分数 += 10
    
    # 6. 相关性风险（10分）
    if portfolio.has_related_sector(stock.sector):
        风险分数 += 10  # 已持有相关板块
    
    return 风险分数
```

**风险决策：**
```
风险评分 < 30：可以买入 ✅
风险评分 30-60：小仓位试探 ⚠️
风险评分 > 60：坚决回避 ❌
```

### 模块4：板块相关性分析（新增）

**作用**：避免重仓高度相关的板块

```python
相关性矩阵：
            新能源车  锂电池  钴锂  光伏  风电
新能源车      1.0    0.85   0.75  0.60  0.50
锂电池        0.85   1.0    0.90  0.55  0.45
钴锂          0.75   0.90   1.0   0.50  0.40
光伏          0.60   0.55   0.50  1.0   0.70
风电          0.50   0.45   0.40  0.70  1.0

相关度>0.7 = 高度相关（避免同时重仓）
相关度0.4-0.7 = 中度相关（控制总仓位）
相关度<0.4 = 低相关（可以分散）
```

**仓位限制：**
```python
规则1：高度相关板块（相关度>0.7）
- 总仓位不超过40%
- 例如：新能源车20% + 锂电池15% = 35% ✅

规则2：中度相关板块（相关度0.4-0.7）
- 总仓位不超过60%

规则3：低相关板块（相关度<0.4）
- 总仓位不超过80%
```

**智能提醒：**
```
⚠️ 警告：您已持有新能源车板块20%
相关板块：锂电池（相关度0.85）、钴锂（相关度0.75）
建议：避免继续加仓相关板块，考虑其他板块分散风险
```

### 模块6：动态止盈止损优化（升级）

**当前问题**：
- 固定止盈止损过于机械
- 没有考虑市场环境
- 没有移动止损

**优化方案：**

```python
def dynamic_stop_loss(
    entry_price,
    current_price,
    market_env,
    sector_stage,
    holding_days
):
    """动态止损"""
    
    盈亏% = (current_price - entry_price) / entry_price * 100
    
    # 1. 基础止损（根据市场环境）
    if market_env == '牛市':
        基础止损 = -8%
    elif market_env == '熊市':
        基础止损 = -5%  # 更严格
    else:
        基础止损 = -6%
    
    # 2. 时间止损（持有时间过长）
    if holding_days > 30 and 盈亏% < 5%:
        return '时间止损：持有超30天无明显盈利'
    
    # 3. 移动止损（盈利>20%后）
    if 盈亏% > 20:
        移动止损价 = current_price * 0.90
        if current_price < 移动止损价:
            return f'移动止损：回调10%，锁定{盈亏%-10:.1f}%利润'
    
    # 4. 板块止损（板块转为衰退期）
    if sector_stage == '衰退期':
        return '板块衰退止损：立即清仓'
    
    # 5. 板块止盈（板块进入爆发期）
    if sector_stage == '爆发期':
        if 盈亏% > 30:
            return '板块爆发止盈：止盈80%'
    
    # 正常持有
    if 盈亏% < 基础止损:
        return f'触发止损：{基础止损}%'
    
    return '继续持有'
```
