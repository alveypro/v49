## 🔧 修复内容

### 1. 修复v5评分器代码错误 ✅

#### 错误1：缺少评分方法
**问题**：v5评分器调用了不存在的 `_score_macd_trend()` 和 `_score_ma_trend()` 方法

**修复**：添加v5专属方法，调用父类v4方法并按比例转换
```python
def _score_macd_trend_v5(self, ind: Dict, close) -> float:
    """MACD趋势（7分）- 从v4的15分按比例转换"""
    score_v4 = self._score_macd_trend_v4(ind, close)
    return min(score_v4 * (7.0 / 15.0), 7.0)

def _score_ma_trend_v5(self, ind: Dict) -> float:
    """均线多头（7分）- 从v4的10分按比例转换"""
    score_v4 = self._score_ma_trend_v4(ind)
    return min(score_v4 * (7.0 / 10.0), 7.0)
```

#### 错误2：止损止盈字段错误
**问题**：访问不存在的 `['price']` 字段

**修复**：使用正确的字段名 `['stop_loss']` 和 `['take_profit']`

## 💡 使用建议

### 适合v5.0启动确认型的场景

✅ **推荐使用：**
- 想要确认趋势后买入
- 追求短期爆发力（3-7天）
- 不想等待潜伏期
- 能够承担适度追高风险

❌ **不推荐使用：**
- 追求极致底部买入
- 长期价值投资（建议用v4.0）
- 大盘暴跌时（风险太大）
- 极度厌恶风险

### v4.0 vs v5.0 如何选择？

| 对比项 | v4.0 潜伏为王 | v5.0 启动确认 |
|-------|-------------|-------------|
| **买入时机** | 启动前潜伏 | 启动后确认 |
| **买入成本** | 更低（底部） | 较高（中位） |
| **等待时间** | 较长（5-10天） | 较短（3-7天） |
| **确定性** | 中等 | 较高 |
| **风险** | 可能继续下跌 | 可能追高回调 |
| **适合人群** | 耐心投资者 | 激进投资者 |

**建议策略：两者结合！**
- 70%资金：v4.0潜伏布局（Tab12）
- 30%资金：v5.0追击确认（Tab13）
- 分散风险，提高胜率
