## 🔍 问题原因分析

### 为什么用户说"选不出股票"？

可能的原因：

#### 1️⃣ 筛选阈值设置过高
```
默认阈值：55分（B级）
符合≥55分：449只股票（16.8%）
符合≥65分：291只股票（10.9%）
符合≥75分：149只股票（5.6%）
```

如果用户设置了70分或80分，就会筛选出很少的股票。

#### 2️⃣ 扫描可能没有成功运行
- 点击"启动扫描"后，需要等待1-2分钟
- 2681只股票需要处理时间
- 如果中途中断，不会有结果

#### 3️⃣ 市值范围设置过窄
```
当前：100-500亿（2681只）
如果设置：200-300亿（可能只有几百只）
```

## 📌 常见问题

### Q1：为什么我的界面没有显示股票？

**A1**：可能原因和解决方法：

1. **阈值太高**：
   - 问题：默认55分，可能筛选太严格
   - 解决：降低到45分或40分

2. **扫描未完成**：
   - 问题：2681只股票需要1-2分钟处理
   - 解决：耐心等待，看进度条

3. **市值范围太窄**：
   - 问题：如设置200-250亿，股票很少
   - 解决：扩大到100-500亿

### Q2：为什么平均分只有27分？

**A2**：这是正常的！

- 当前市场整体处于调整状态
- 4维评分体系技术面主导（61%）
- 需要同时满足：放量上涨+均线多头+MACD金叉
- 27分的平均分说明大部分股票只满足1-2个维度
- 但仍有29%的股票（775只）≥45分

### Q3：为什么很多股票0分？

**A3**：可能触发了"一票否决"：

- **放量下跌**：量比>1.5 且 3日跌>2%
- **这是主力出货的信号，直接0分**
- 或者所有维度都不符合条件

### Q4：CSV文件有重复？

**A4**：可能是扫描时数据库中有重复记录，属于正常现象。实际使用时按ts_code去重即可。
