## 🐛 问题描述

用户反馈：
> **"60分都找不到股票，说明肯定有问题"**

**实际情况**:
- v6.0策略扫描了2778只股票（50-800亿市值）
- 评分阈值60分
- **结果：找不到任何≥60分的股票**

---

## 🔍 根本原因分析

经过深入检查，发现了**两个严重的调用错误**：

### 1. ⚠️⚠️⚠️ v6.0缺少`ts_code`参数（核心问题）

#### 错误代码

```python
# ❌ 错误：只传了stock_data，缺少ts_code
score_result = vp_analyzer.evaluator_v6.evaluate_stock_v6(stock_data)
```

#### 正确定义

```python
# v6.0评分器需要两个参数
def evaluate_stock_v6(self, stock_data: pd.DataFrame, ts_code: str) -> Dict:
    # ts_code用于获取外部数据
```

#### 影响分析

v6.0需要`ts_code`来获取以下数据：

| 维度 | 分值 | 数据来源 | 缺少ts_code的结果 |
|------|------|---------|------------------|
| **板块热度** | 25分 | Tushare行业数据 | ❌ 0分 |
| **资金流向** | 20分 | Tushare资金流 | ❌ 0分 |
| **相对强度** | 10分 | 大盘对比 | ❌ 0分 |
| **技术突破** | 20分 | 本地历史数据 | ✅ 正常 |
| **短期动量** | 15分 | 本地历史数据 | ✅ 正常 |
| **量能配合** | 5分 | 本地历史数据 | ✅ 正常 |
| **筹码结构** | 3分 | 本地历史数据 | ✅ 正常 |
| **安全边际** | 2分 | 本地历史数据 | ✅ 正常 |
| **总分** | 100分 | - | ❌ 最多45分 |

**结论**：缺少`ts_code`导致**55分直接丢失**，最高只能得45分，无法达到60分阈值！

### 2. ⚠️ v5.0方法名错误

#### 错误代码

```python
# ❌ 错误：方法名不存在
score_result = vp_analyzer.evaluator_v5.evaluate_stock_v5(stock_data)
```

#### 正确方法名

```python
# ✅ 正确：v5.0继承自v4.0，方法名仍然是evaluate_stock_v4
score_result = vp_analyzer.evaluator_v5.evaluate_stock_v4(stock_data)
```

---

## ✅ 修复方案

### 修复1：v6.0传入ts_code

```python
# 修复前（4214行）
score_result = vp_analyzer.evaluator_v6.evaluate_stock_v6(stock_data)

# 修复后
score_result = vp_analyzer.evaluator_v6.evaluate_stock_v6(stock_data, ts_code)  # ← 传入ts_code
```

### 修复2：v5.0使用正确方法名

```python
# 修复前（4976行）
score_result = vp_analyzer.evaluator_v5.evaluate_stock_v5(stock_data)

# 修复后
score_result = vp_analyzer.evaluator_v5.evaluate_stock_v4(stock_data)  # ← 正确的方法名
```

### 修复3：优化v6.0参数和提示

```python
# 调整默认评分阈值
# 修复前：75分（太高）
# 修复后：60分（合理）

# 添加数据依赖说明
st.warning("""
⚠️ **v6.0超短线策略数据依赖说明**：
- **板块热度**（25分）：需要查询行业数据
- **资金流向**（20分）：需要Tushare Pro高级接口（2000+积分）
- **技术指标**（55分）：使用本地历史数据

💡 **建议**：
- 如果Tushare积分不足，板块和资金流维度可能为0分
- 建议从60分开始扫描（而不是75分）
- 或使用v4.0/v5.0策略（不依赖高级接口）
""")
```
