# ✅ 市值参数修复完成

**问题**: `streamlit.errors.StreamlitValueBelowMinError: The value 500 is less than the min_value 1000.`  
**原因**: v5.0/v6.0的市值上限默认值可能小于用户设置的最小值  
**状态**: ✅ **已修复**

---

## 🔴 问题分析

### 错误信息

```
StreamlitValueBelowMinError: The value 500 is less than the min_value 1000.

File "/Users/mac/QLIB/终极量价暴涨系统_v49.0_长期稳健版.py", line 5168
    cap_max_v5 = st.number_input(...)
```

### 问题根源

**Streamlit的number_input要求**：`value >= min_value`

**v5.0的配置问题**：
```python
cap_min_v5 = st.number_input(value=100, ...)  # 用户可以改成1000
cap_max_v5 = st.number_input(
    min_value=cap_min_v5,  # 如果cap_min_v5=1000
    value=500,              # ❌ 500 < 1000，报错！
    ...
)
```

**发生场景**：
1. 用户之前把`cap_min_v5`设置为1000亿
2. Streamlit记住了这个值（session state）
3. 下次启动时，`cap_max_v5`的`min_value=1000`，但`value=500`
4. 报错：500 < 1000

---

## ✅ 修复方案

### 修复1：v5.0市值上限

**修复前**：
```python
cap_max_v5 = st.number_input(
    "最大市值（亿元）",
    min_value=cap_min_v5,
    max_value=10000,
    value=500,  # ❌ 可能小于cap_min_v5
    step=50,
    help="建议100-500亿，中等市值爆发力强",
    key="cap_max_v5"
)
```

**修复后**：
```python
cap_max_v5 = st.number_input(
    "最大市值（亿元）",
    min_value=cap_min_v5,
    max_value=10000,
    value=max(1500, cap_min_v5),  # ✅ 确保 value >= min_value
    step=50,
    help="建议100-1500亿，中等市值爆发力强",
    key="cap_max_v5"
)
```

**逻辑**：
- 如果`cap_min_v5 <= 1500`：使用`value=1500`
- 如果`cap_min_v5 > 1500`：使用`value=cap_min_v5`
- 保证`value >= min_value`

## 💡 为什么会出现这个问题

### Streamlit的Session State机制

**Streamlit会记住用户的输入**：
1. 用户第一次设置`cap_min_v5 = 1000`
2. Streamlit保存到session state
3. 下次启动时，`cap_min_v5`从session恢复为1000
4. 但代码中`cap_max_v5`的`value`是硬编码的500
5. 导致`min_value (1000) > value (500)` → 报错

### 为什么v4.0和v7.0没问题

**v4.0和v7.0的设计**：
```python
cap_min = 0  # 默认0
cap_max = 0  # 默认0，表示不限制

# 所以永远满足 value (0) >= min_value (0)
```

### 为什么v5.0和v6.0有问题

**v5.0和v6.0的设计**：
```python
cap_min = 100   # 动态，用户可改
cap_max = 500   # 固定，硬编码

# 如果用户改cap_min为600，就会导致：
# value (500) < min_value (600) → 报错
```

## 🎯 最佳实践

### 对于动态min_value的number_input

**❌ 错误做法**：
```python
min_val = st.number_input("最小值", value=100)
max_val = st.number_input(
    "最大值", 
    min_value=min_val,
    value=500  # ❌ 硬编码，可能小于min_val
)
```

**✅ 正确做法**：
```python
min_val = st.number_input("最小值", value=100)
max_val = st.number_input(
    "最大值", 
    min_value=min_val,
    value=max(500, min_val)  # ✅ 动态调整，确保 >= min_val
)
```

## 🎊 总结

### 问题

v5.0和v6.0的市值上限默认值可能小于最小值，导致Streamlit报错。

### 根因

`number_input`的`value`参数硬编码，没有考虑动态的`min_value`。

### 修复

使用`max(default_value, min_value)`动态调整`value`，确保总是满足`value >= min_value`。

### 效果

✅ v5.0和v6.0现在可以正常启动，不会再报`StreamlitValueBelowMinError`错误！
