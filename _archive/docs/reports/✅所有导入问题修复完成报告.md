## 🎯 根本原因

### Python变量作用域规则

当Python在函数中发现对某个名字的**赋值操作**时（包括import），会将该名字视为**局部变量**。

```python
# 全局
import sqlite3

def main():
    conn = sqlite3.connect(...)  # ❌ 错误！
    # Python认为sqlite3是局部变量，但还未赋值
    
    import sqlite3  # 局部导入在后面
    # 此时sqlite3才被赋值
```

### 正确做法

**方法1: 只在顶部导入（推荐）**
```python
# 文件顶部
import sqlite3
from datetime import datetime

def main():
    conn = sqlite3.connect(...)  # ✅ 正确
    time = datetime.now()         # ✅ 正确
```

**方法2: 如果必须局部导入，在使用前导入**
```python
def main():
    import sqlite3  # 在使用前导入
    conn = sqlite3.connect(...)  # ✅ 正确
```

## 💡 经验总结

### 避免类似问题的最佳实践

1. **统一在文件顶部导入**
   ```python
   # ✅ 推荐
   import sqlite3
   import pandas as pd
   from datetime import datetime
   
   def main():
       # 直接使用
   ```

2. **避免在函数内部重复导入**
   ```python
   # ❌ 不推荐
   def main():
       import sqlite3  # 创建局部变量
   ```

3. **如果必须局部导入，确保在使用前导入**
   ```python
   # ⚠️ 可以但不推荐
   def main():
       import sqlite3  # 在最前面
       conn = sqlite3.connect(...)  # 然后使用
   ```

4. **使用IDE的静态检查**
   - PyCharm会警告UnboundLocalError
   - VS Code with Pylance也会提示
