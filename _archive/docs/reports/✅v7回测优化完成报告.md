for i in range(60, len(stock_data) - holding_days):
    current_data = stock_data.iloc[:i+1].copy()
    
    # ✅ 确保列名一致性 - 只在需要时转换

        current_data = current_data.rename(columns={'close': 'close_price'})
    
    # ... 评分逻辑 ...
    
    # ✅ 计算未来收益 - 使用正确的列名
    close_col = 'close' if 'close' in stock_data.columns else 'close_price'
    buy_price = stock_data.iloc[i][close_col]
    sell_price = stock_data.iloc[i + holding_days][close_col]

## 💡 策略使用建议

### v7.0 策略的独特性

**v7 ≠ v4加强版**，它是完全不同的系统：

#### v4.0 (潜伏为王)
- 固定权重
- 底部价值导向
- 评分相对宽松
- 60分 = 合格标的

#### v7.0 (终极智能)
- **动态权重** - 根据市场环境调整
- **行业轮动** - 识别热门行业加分
- **三层过滤** - 市场情绪+行业景气+资金流向
- **评分更严** - 60分 = v4的65-70分水平

### 为什么v7需要60分阈值？

1. **动态权重影响**：
   - 牛市：主力行为权重高
   - 熊市：底部特征权重高
   - 震荡市：量价配合权重高
   - **同样的股票，不同环境分数不同**

2. **三层过滤机制**：
   - 第一层：市场情绪筛选
   - 第二层：行业景气度筛选
   - 第三层：资金流向筛选
   - **已经做了多次过滤，60分股票质量很高**

3. **行业加分效应**：
   - 热门行业+5-10分
   - 如果设置70分阈值，会错过很多好股票
   - **60分 + 行业加分 = 实际65-70分质量**
