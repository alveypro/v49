### P0-003: 数据库连接资源泄漏

#### 问题详情

**现状**:
- 多处代码在异常情况下未正确关闭数据库连接
- 可能导致连接耗尽和"database is locked"错误

**问题代码示例**:
```python
# 主系统第4518行
conn = sqlite3.connect(PERMANENT_DB_PATH)

if stocks_df.empty:
    st.error("...")
    conn.close()  # ✅ 这里关闭了
else:
    # 长时间处理...
    # 如果这里抛出异常怎么办？❌
    conn.close()  # 可能不会执行到
```

#### 修复方案

使用上下文管理器确保连接总是被关闭：

**修复位置1** - 第4514-4650行（v4.0策略扫描）:
```python
# 修改前
conn = sqlite3.connect(PERMANENT_DB_PATH)
try:
    stocks_df = pd.read_sql_query(query, conn)
    # ... 处理 ...
    conn.close()
except Exception as e:
    conn.close()  # ❌ 可能忘记关闭
    raise

# 修改后
try:
    with sqlite3.connect(PERMANENT_DB_PATH) as conn:
        stocks_df = pd.read_sql_query(query, conn)
        # ... 所有数据库操作 ...
        # conn会自动关闭，即使异常也会关闭 ✅
except Exception as e:
    logger.error(f"错误: {e}")
    st.error(f"❌ 扫描失败: {e}")
```

**需要修复的位置**:
1. 第4518行 - v4.0策略扫描
2. 第4891行 - v5.0策略扫描  
3. 第5238行 - v6.0策略扫描
4. 第5941行 - 回测系统
5. 所有其他使用 `sqlite3.connect()` 的地方
