## 🎓 核心技术说明

### 信号识别算法

系统使用4种经典量价信号：

#### 1. 底部放量
```python
条件：
- 价格位于近期底部（价格位置 < 30%）
- 成交量显著放大（近3日均量 > 历史均量 × 1.5）

含义：主力可能在底部吸筹
```

#### 2. 量价背离
```python
条件：
- 价格下跌 > 3%
- 成交量放大 > 30%

含义：下跌但放量，可能是主力震仓
```

#### 3. 缩量蓄势突破
```python
条件：
- 近期波动率低（横盘）
- 前期缩量（成交量萎缩 > 20%）
- 最近放量（成交量放大 > 30%）

含义：横盘整理后放量突破
```

#### 4. 平台突破
```python
条件：
- 突破近期平台高点 > 2%
- 温和放量（成交量放大 > 20%）

含义：突破压力位，趋势可能延续
```

### 回测验证机制

**核心思想**：不是预测，而是验证

1. **历史信号识别**
   - 在历史数据上识别所有买入信号
   - 记录信号触发时的价格

2. **实际收益计算**
   ```python
   future_return = (未来N日价格 / 当前价格) - 1
   ```
   - 使用真实历史成交价
   - 不做任何假设或预测

3. **策略评估**
   - 统计所有信号的实际收益
   - 计算胜率、平均收益
   - 评估策略有效性

4. **当前应用**
   - 只推荐触发相同信号的股票
   - 预期收益 = 历史相同信号的平均收益
   - 概率保障而非确定性预测

## 🔧 常见问题解答

### Q1: 为什么回测收益和预期不一样？
**A**: 因为v36使用的是**真实历史数据**，不是理想化的模拟。真实市场存在各种不确定性，所以回测结果会有波动。这正是系统的价值所在——告诉你真实的历史表现。

### Q2: 如何确认数据是最新的？
**A**: 
1. 进入"🔄 实时更新"标签页
2. 点击"🔍 检查数据库状态"
3. 看到"✅ 数据已是最新"即可
4. 如果不是最新，运行一次快速更新

### Q3: 参数优化需要多久？
**A**: 取决于测试组合数量：
- 少量组合（9个）：3-5分钟
- 中等组合（27个）：8-12分钟
- 大量组合（45个）：15-20分钟

建议先用少量组合测试，找到大致范围后再细化。

### Q4: 快速更新和全量更新的区别？
**A**: 
- **快速更新**：只更新当前推荐的股票（通常10-50只），速度快
- **全量更新**：更新数据库所有股票（3000+只），确保数据完整

日常使用快速更新即可，每周做一次全量更新。

### Q5: 为什么有些股票没有预期收益？
**A**: 可能原因：
1. 该股票在历史回测中没有触发过相同信号
2. 信号类型较新，历史样本不足
3. 数据不完整

这种情况下可以参考信号强度来判断。

### Q6: 推荐的股票一定会涨吗？
**A**: **不一定**。系统只是基于历史数据告诉你：
- 相同信号在历史上的表现
- 平均来说有多少概率上涨
- 平均收益是多少

投资有风险，请结合基本面分析和个人判断。

### Q7: 如何提高选股准确率？
**A**: 建议：
1. 提高信号强度阈值（如从40提高到60）
2. 只选择预期收益 > 3%的股票
3. 行业分散，不要集中
4. 结合"回测详情"查看收益分布
5. 定期重新优化参数

### Q8: 数据更新失败怎么办？
**A**: 可能原因和解决方案：
1. **Tushare积分不足**：升级账号或降低请求频率
2. **网络连接问题**：检查网络，稍后重试
3. **请求频率过高**：增加更新间隔时间
4. **非交易日**：等到交易日再更新

### Q9: 可以保存优化结果吗？
**A**: 当前版本暂不支持自动保存。建议：
1. 截图保存最佳参数
2. 手动记录到文档
3. 下次使用时手动输入

未来版本会添加参数保存功能。

### Q10: 系统支持哪些市场？
**A**: 目前支持：
- A股主板
- 中小板
- 创业板
- 科创板

数据来源于Tushare Pro，覆盖全A股市场。
