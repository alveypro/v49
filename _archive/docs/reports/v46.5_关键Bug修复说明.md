## 🔥 **根本原因（关键Bug）**

### 代码位置：第338-339行
```python
if normalized_score >= threshold_score:
    signal_info = {
        ...
        'momentum': round(float(indicators.get('momentum_5', 0)), 2),
        'ma_score': 20 if ma5 > ma10 > ma20 > 0 and close[-1] > ma5 else ...,
        #                   ↑ 这里使用了ma5变量
        'volume_price_score': 25 if normalized_score >= 25 else ...
    }
    signals.append(signal_info)  # ❌ 这里报错但被except吞掉了！
```

### 问题分析

1. **条件评分时**：
```python
if indicators:  # ✅ 只有indicators不为空才定义ma5
    ma5 = indicators.get('ma5', 0)
    ma10 = indicators.get('ma10', 0)
    ma20 = indicators.get('ma20', 0)
    # ... 评分逻辑
```

2. **append signal_info时**：
```python
'ma_score': 20 if ma5 > ma10 > ma20 > 0 ...
#                   ↑ 但这里直接使用ma5！
#                   ↑ 如果indicators为空，ma5未定义！
#                   ↑ NameError: name 'ma5' is not defined
```

3. **异常处理**：
```python
except Exception as e:
    pass  # ❌ 把NameError吞掉了！
```

### 为什么indicators经常为空？

`_calculate_technical_indicators`计算失败的情况非常常见：
- 数据不足
- 数据异常
- talib计算失败
- 等等

**估计80-90%的股票indicators都是空的！**

## ✅ **修复方案**

### 核心修复
```python
# ❌ 修复前：直接使用ma5（可能未定义）
'ma_score': 20 if ma5 > ma10 > ma20 > 0 and close[-1] > ma5 else ...

# ✅ 修复后：安全获取
safe_ma5 = indicators.get('ma5', 0) if indicators else 0
safe_ma10 = indicators.get('ma10', 0) if indicators else 0
safe_ma20 = indicators.get('ma20', 0) if indicators else 0
safe_momentum = indicators.get('momentum_5', 0) if indicators else 0

'ma_score': 20 if safe_ma5 > safe_ma10 > safe_ma20 > 0 and close[-1] > safe_ma5 else ...
'momentum': round(float(safe_momentum), 2)
```

### 为什么之前v46.1能选出股票？

v46.1的代码中有：
```python
if not indicators:
    continue  # 如果indicators失败就跳过这只股票
```

这样虽然跳过了很多股票，但**至少对于indicators成功的股票**，ma5等变量是有定义的，不会报错。

而v46.5中，为了不跳过股票，移除了这个限制，但**忘记了在append时也要安全处理**！

## 💡 **经验教训**

### 1. 异常处理不能太粗暴
```python
# ❌ 错误示范
except Exception as e:
    pass  # 吞掉所有错误，无法调试
```

```python
# ✅ 正确做法
except Exception as e:
    logger.error(f"处理失败: {e}")  # 至少记录日志
    logger.error(traceback.format_exc())  # 打印堆栈
```

### 2. 变量作用域要注意
```python
if condition:
    var = some_value  # var只在if内有效
    
# 这里使用var可能未定义
result = var  # ❌ NameError
```

### 3. 日志是调试的好帮手
```python
# 打印中间状态
logger.info(f"[样本] score={score}, ...")  # 帮助发现"有分数却选不出"

# 打印最终结果
logger.info(f"🎉 成功返回 {len(signals)} 条信号！")  # 确认是否真的返回了
```
