## 1. 精度优先优化（核心）

### 1.1 统一回测口径
- 统一回测数据窗口长度（默认 420 交易日）
- 统一缺失值处理：
  - 价格/成交量列统一 `ffill()`
  - 异常值统一截断
- 统一涨跌幅计算方法与字段名

### 1.2 统一信号阈值校准方法
- 把 v4~v9 的阈值校准统一成：
  - 目标胜率 / 风险回撤控制
  - 统一评分区间及评分映射规则

### 1.3 市场环境过滤增强
- 强化「市场环境」在评分中的权重（对 v7/v8/v9/组合）
- 对 v4/v5 加入“弱市”扣分机制

### 1.4 策略一致性约束（减少噪音）
- 统一启用多周期一致性过滤
- 对异常分数（极端高/低）增加惩罚

---

## 2. 速度优化（不影响精度）

### 2.1 批量拉取 + 向量化
- 将单只股票 SQL 查询改为批量查询
- 指标计算尽量向量化，减少 Python 循环

### 2.2 中间特征缓存
- 统一生成一次特征（如 120 日均线、ATR、RSI）
- 多策略共享特征缓存

### 2.3 扫描结果缓存
- v4~v9 + 组合扫描结果写入缓存表或 CSV
- 页面读取缓存优先，减少实时计算

---

## 3. 稳定性优化

### 3.1 数据库稳定性
- SQLite 全面启用 WAL + busy_timeout
- 所有连接统一封装

### 3.2 异常容错统一
- 评分器异常返回空，不中断全局
- 资金表缺失时自动降级逻辑

### 3.3 日志规范化
- 统一日志输出目录
- 重要异常输出到独立文件

---

## 4. 资源占用优化（最后级别）

- 限制并发线程数（避免 CPU 饱和）
- 限制扫描数量只作为可选开关

---

## 5. 策略级优化要点

### v4/v5
- 保持现有逻辑，统一阈值校准
- 强化弱市过滤

### v6
- 资金与板块热度缺失时自动降级
- 增加行业因子阈值约束

### v7/v8
- 市场环境权重提高
- 抽取统一的「市场状态」特征

### v9
- 回测窗口统一
- 市值和成交额过滤口径统一

### 组合策略
- 统一各策略输出口径
- 提高一致性权重，降低波动

### 稳定上涨
- 强化趋势过滤
- 回撤限制更严格

### AI
- 模型参数更新频率降低
- 只对稳定样本做进化
