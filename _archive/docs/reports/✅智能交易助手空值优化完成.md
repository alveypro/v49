### 优化4: UI显示时安全获取字段值

**文件**: `终极量价暴涨系统_v49.0_长期稳健版.py`  
**位置**: 智能交易助手Tab - 持仓显示

```python
for idx, holding in holdings.iterrows():
    profit_color = "🟢" if holding['profit_loss'] > 0 else "🔴"  # ❌ 可能是None
    
    with st.expander(f"{holding['stock_name']} - {holding['profit_loss_pct']*100:.2f}%"):
        st.metric("盈亏", f"¥{holding['profit_loss']:.2f}")  # ❌ 可能是None

for idx, holding in holdings.iterrows():
    # ✅ 安全获取盈亏值，默认为0
    profit_loss = holding.get('profit_loss', 0) or 0
    profit_loss_pct = holding.get('profit_loss_pct', 0) or 0
    buy_price = holding.get('buy_price', 0) or 0
    current_price = holding.get('current_price', 0) or buy_price

    with st.expander(f"{holding['stock_name']} - {profit_loss_pct*100:.2f}%"):
        st.metric("盈亏", f"¥{profit_loss:.2f}")  # ✅ 安全显示
        
    # ✅ 其他字段也安全处理
    cost_total = holding.get('cost_total', 0) or 0
    current_value = holding.get('current_value', 0) or 0
```

## 💡 设计原则

### 防御性编程

1. **永远不信任数据库数据**
   - 使用`.get()`方法获取字典值
   - 提供默认值
   - 使用`or 0`处理None

2. **比较前检查类型**
   ```python
   if value is not None and pd.isna(value) == False:
       # 安全比较
   ```

3. **数据库使用COALESCE**
   ```sql
   COALESCE(current_price, buy_price)  -- 优先用current_price，没有则用buy_price
   ```

4. **除法前检查除数**
   ```python
   result = a / b if b > 0 else 0
   ```
