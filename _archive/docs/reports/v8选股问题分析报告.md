## 🔬 根本原因分析

### 原因1：v8实际使用的是v4评分器（v7加载失败）

```python
# v8初始化日志显示：
⚠️ v7评分器加载失败，将使用v4作为基础
```

**影响**：
- v4是"潜伏为王"策略，评分标准很严格
- v4平均分仅32.14分，远低于及格线
- v4分占v8总分的60%权重，严重拖累v8

### 原因2：v8的评分公式过于依赖v7

```python
# v8评分公式
final_score = 0.6 * v7_score + 0.4 * advanced_score

# 实际计算
v8_score = 0.6 * 32.14 + 0.4 * 50.00 = 19.28 + 20.00 = 39.28分
```

**问题**：
- v7权重60%太高
- 即使高级因子满分100，v8最高也只能到80分：
  - v8 = 0.6 * 0 + 0.4 * 100 = 40分 (v7为0时)
  - v8 = 0.6 * 50 + 0.4 * 100 = 70分 (v7为50时)
  - v8 = 0.6 * 100 + 0.4 * 100 = 100分 (v7为100时)

### 原因3：v4评分器本身的特点

v4是"潜伏为王"版，专注于：
- 底部特征（20分）
- 潜伏价值（20分）
- 即将启动但未启动的股票

**当前市场环境可能不符合v4的严格标准**

## ✅ 结论

### 问题类型
**评分计算正确，但基础评分过低**

v8的代码逻辑没有错误，问题在于：
1. ✅ **不是策略出错** - v8的高级因子、市场过滤、ATR风控都正常
2. ✅ **不是评分出错** - 计算公式正确（0.6*v7 + 0.4*advanced）
3. ❌ **是基础评分太低** - v7(实际v4)平均32分，拖累v8总分

### 核心原因
**v7评分器加载失败，降级使用了v4，而v4评分太严格**

### 最佳解决方案
1. **首选**：修复v7评分器（让v8使用真正的v7）
2. **次选**：调整v8权重（降低v7权重，提高高级因子权重）
3. **备选**：降低推荐阈值到50-55分（临时方案）
