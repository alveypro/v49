### 修复2: v6.0缺少ts_code参数 ✅

**位置**: 第2309-2317行  
**文件**: `终极量价暴涨系统_v49.0_长期稳健版.py`

**修改前**:
```python
# 遍历可能的买入点
for i in range(30, len(stock_data) - holding_days - 1):
    current_data = stock_data.iloc[:i+1].copy()
    
    # 调用对应版本的评分方法
    eval_result = getattr(evaluator, eval_method)(current_data)  # ❌ v6缺少ts_code
    
    if not eval_result['success']:
        continue
```

**修改后**:
```python
# 遍历可能的买入点
for i in range(30, len(stock_data) - holding_days - 1):
    current_data = stock_data.iloc[:i+1].copy()
    
    # 调用对应版本的评分方法（v6需要传递ts_code）
    if version == 'v6':
        eval_result = getattr(evaluator, eval_method)(current_data, ts_code)  # ✅ 传递ts_code
    else:
        eval_result = getattr(evaluator, eval_method)(current_data)
    
    if not eval_result['success']:
        continue
```

**原因**: v6.0的evaluate_stock_v6()方法签名为`evaluate_stock_v6(stock_data, ts_code)`，需要两个参数，核心策略中心第5298行也是这样调用的。

**效果**: v6.0回测现在正确传递ts_code参数，与核心策略中心完全一致。
