## 🔥 问题分析：为什么找不到≥60分的股票？

### 问题1：板块热度（25分）- 标准过于严格 ⚠️⚠️⚠️

#### 评分标准

```python
# 板块3天涨幅（12分）
if sector_change > 5:    # 强势板块
    score += 12
elif sector_change > 3:
    score += 9
elif sector_change > 1:
    score += 5
elif sector_change > 0:
    score += 2

# 板块排名（8分）
if rank <= 5:    # 前5名
    score += 8
elif rank <= 10: # 前10名
    score += 6
elif rank <= 20: # 前20名
    score += 3
```

#### 问题分析

**市场现实**：
- 当前市场环境（震荡市）
- 大部分板块3天涨幅在-2%到+2%之间
- 只有极少数热点板块3天涨幅>5%

**举例**：
```
假设当前市场：
- 90%的板块：3天涨幅 -2% ~ +2%（得分：0-5分）
- 8%的板块：3天涨幅 +2% ~ +5%（得分：5-9分）
- 2%的板块：3天涨幅 >5%（得分：12分）

结果：
- 90%的股票在板块热度上只能得0-5分
- 而板块热度满分25分！
```

**建议调整**：
```python
# 调整后的标准（更符合实际）
if sector_change > 3:    # 降低要求
    score += 12
elif sector_change > 1.5:
    score += 9
elif sector_change > 0.5:
    score += 5
elif sector_change > -1:  # 不跌就给分
    score += 2
```

### 问题2：资金流向（20分）- 依赖高级接口 ⚠️⚠️⚠️

#### 评分标准

```python
# 需要Tushare Pro接口（2000+积分）
money_flow = self.data_provider.get_money_flow(ts_code, days=3)

# 大单净流入（10分）
if consecutive_days >= 3:    # 连续3天流入
    score += 10

    score += 7

# 超大单（5分）
if elg_net > 3000:     # >3000万
    score += 5

# 北向资金（5分）
if consecutive_buy_days >= 3:
    score += 5
```

#### 问题分析

**数据获取问题**：
```python
# v6_data_provider.py的容错机制
try:
    money_flow = self.pro.moneyflow(ts_code=ts_code, ...)
except:
    # 积分不足时返回默认值
    return {
        'net_mf_amount': 0,          # ← 0分
        'buy_elg_amount': 0,         # ← 0分
        'consecutive_inflow_days': 0 # ← 0分
    }
```

**影响**：
- 如果用户Tushare积分不足
- **资金流向这20分全部为0！**
- 这是很大的损失

**建议调整**：
1. 降低对资金流的依赖
2. 或者使用本地可计算的指标代替

### 问题3：技术突破（20分）- vol_ratio要求过高 ⚠️⚠️

#### 评分标准

```python
# 突破关键位（10分）
if current_price > recent_high and vol_ratio > 1.5:
    score += 10    # 需要放量1.5倍
elif current_price > ma20 and current_price > ma60 and vol_ratio > 1.3:
    score += 7     # 需要放量1.3倍
elif current_price > ma20 and vol_ratio > 1.2:
    score += 4     # 需要放量1.2倍

# 放量确认（5分）
if vol_ratio > 2.0:
    score += 5
elif vol_ratio > 1.5:
    score += 3
elif vol_ratio > 1.2:
    score += 1
```

#### 问题分析

**市场现实**：
```
日常市场统计：
- 70%的股票：vol_ratio = 0.8 ~ 1.1（缩量或平量）
- 25%的股票：vol_ratio = 1.1 ~ 1.5（温和放量）
- 5%的股票：vol_ratio > 1.5（明显放量）

按当前标准：
- 70%的股票在技术突破上得0分！
- 25%的股票最多得1-4分
- 只有5%的股票能得高分
```

**建议调整**：
```python
# 降低vol_ratio要求
if current_price > recent_high and vol_ratio > 1.0:  # 从1.5降到1.0
    score += 10
elif current_price > ma20 and current_price > ma60 and vol_ratio > 0.9:
    score += 7
elif current_price > ma20:  # 不要求放量
    score += 4

# 放量确认
if vol_ratio > 1.5:
    score += 5
elif vol_ratio > 1.2:
    score += 3
elif vol_ratio > 1.0:  # 从1.2降到1.0
    score += 1
```

```python
relative_strength = stock_change - market_change
```

#### 问题分析

```

- 要跑赢5%，股票需要涨5%（很难）
- 要跑赢3%，股票需要涨3%
- 要跑赢1%，股票需要涨1%

实际情况：
- 60%的股票：相对强度 -3% ~ +1%（得分：0-4分）
- 30%的股票：相对强度 +1% ~ +3%（得分：4-7分）
- 10%的股票：相对强度 >3%（得分：7-10分）
```

**建议调整**：
```python
if relative_strength > 3:    # 从5%降到3%
    score += 10
elif relative_strength > 1.5: # 从3%降到1.5%
    score += 7
elif relative_strength > 0:   # 跑赢就给分
    score += 4
```

### 问题6：风险扣分（0-30分）- 扣分过重 ⚠️⚠️

#### 评分标准

```python
# 价格位置扣分
if price_pos >= 0.80:
    penalty += 15    # 极高位扣15分
elif price_pos >= 0.70:
    penalty += 10    # 高位扣10分
elif price_pos >= 0.60:
    penalty += 5     # 偏高位扣5分

# 短期暴涨扣分
if change_3d > 20:
    penalty += 10
elif change_3d > 15:
    penalty += 5
```

#### 问题分析

**矛盾**：
- **短期动量维度**要求：3天涨幅>10%得满分（+10分）
- **风险扣分**：3天涨幅>15%扣5分，>20%扣10分
- **结果**：短期暴涨的股票，加分又被扣掉！

**举例**：
```
股票A：3天涨18%
- 短期动量得分：+10分（满分）
- 风险扣分：-5分（涨幅>15%）
- 净得分：+5分（亏了！）

股票B：3天涨3%
- 短期动量得分：+4分
- 风险扣分：0分
- 净得分：+4分（差不多）
```

**建议调整**：
```python
# 减少风险扣分力度
if price_pos >= 0.85:    # 从0.80提高到0.85
    penalty += 10        # 从15分降到10分
elif price_pos >= 0.75:  # 从0.70提高到0.75
    penalty += 5         # 从10分降到5分

# 短期暴涨扣分
if change_3d > 25:       # 从20%提高到25%
    penalty += 8         # 从10分降到8分
elif change_3d > 18:     # 从15%提高到18%
    penalty += 3         # 从5分降到3分
```

## 🎯 核心结论

### 为什么找不到≥60分的股票？

1. **板块热度（25分）**：
   - ❌ 标准过严，要求板块3天涨>5%
   - 📊 实际情况：90%板块只能得0-5分

2. **资金流向（20分）**：
   - ❌ 依赖Tushare高级接口（2000+积分）
   - 📊 积分不足时：这20分全部为0！

3. **技术突破（20分）**：
   - ❌ 要求vol_ratio>1.2才有分
   - 📊 实际情况：70%股票vol_ratio<1.2

4. **短期动量（15分）**：
   - ❌ 要求3天涨>10%才满分
   - 📊 实际情况：95%股票3天涨幅<10%

5. **相对强度（10分）**：
   - ❌ 要求跑赢大盘5%才满分
   - 📊 实际情况：90%股票跑赢<3%

6. **风险扣分（0-30分）**：
   - ❌ 扣分过重
   - ❌ 与短期动量矛盾（涨得多反而被扣分）

```python
# 用本地指标代替资金流向（20分）
def _score_money_flow_local(self, indicators, pct_chg, volume):
    """不依赖Tushare高级接口的资金流评分"""
    score = 0
    
    # 1. 量能强度（10分）

        score += 10
    elif vol_ratio > 1.2:
        score += 7

    if recent_change > 5 and vol_ratio > 1.2:
        score += 10
    elif recent_change > 0 and vol_ratio > 1.0:
        score += 6
    
    return min(score, 20)
```
