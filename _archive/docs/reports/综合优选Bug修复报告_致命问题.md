**Bug代码**：

**为什么没有报错？**
```python

    # ... 扫描代码 ...
    综合结果.append({...})  # 这里抛出NameError
```

1. 每只符合条件的股票都会在5749行抛出异常

4. 直接跳过这只股票，不添加到结果列表
5. 最终结果列表为空，用户看不到任何股票！

## 🔍 为什么这个Bug这么隐蔽？

### 1. 静默失败
```python
except Exception as e:
    continue  # 不打印错误，直接跳过
```
- 没有`print(e)`或`logger.error(e)`
- 用户看不到任何错误信息
- 只看到"扫描完成，0只股票"

### 2. 评分逻辑正常
- `evaluate_stock_ultimate_fusion`函数内部有自己的`close`变量
- 评分逻辑完全正常，返回正确的分数
- 问题出在外层的结果组装代码

### 3. 离线脚本正常
- 离线脚本有正确的代码
- 测试时使用离线脚本，发现一切正常
- 误以为界面版本也正常

### 4. 变量作用域问题
- `close`在评分函数内部是局部变量
- 评分函数返回后，`close`变量不存在
- 外层代码需要重新提取

## 💡 经验教训

### 1. 不要静默失败
**错误做法**：
```python
except Exception as e:
    continue  # ❌ 静默失败
```

**正确做法**：
```python
except Exception as e:
    logger.error(f"处理{ts_code}失败: {e}")  # ✅ 记录错误
    continue
```

### 2. 代码一致性
- 离线脚本和界面版本应该使用相同的代码
- 或者提取为共用函数
- 避免代码重复导致的不一致

### 3. 变量作用域
- 注意变量的作用域
- 函数内部的局部变量不能在外部使用
- 需要时重新提取或传递

### 4. 充分测试
- 不仅测试离线脚本
- 也要测试界面功能
- 确保所有路径都正常工作
