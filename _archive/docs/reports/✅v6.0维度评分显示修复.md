# ✅ v6.0维度评分显示修复完成

## 🔍 问题发现

用户反馈推荐股票列表中：
- **综合评分正常**：60-76分
- **所有维度评分都是0.0**：板块热度、资金流向、技术突破等8个维度全部显示0.0

这明显不合理，因为综合评分是由各维度评分计算出来的！

## 🎯 问题定位

**根本原因**：字段名不匹配！

1. **v6评分器返回**（`comprehensive_stock_evaluator_v6.py`）：
```python
result = {
    'dim_scores': dim_scores,  # ← 字段名是dim_scores
    ...
}
```

2. **主系统提取**（`终极量价暴涨系统_v49.0_长期稳健版.py`）：
```python
dim_scores = score_result.get('dimension_scores', {})  # ← 错误！取不到值
```

由于字段名不匹配，`get('dimension_scores', {})`返回空字典，导致所有维度评分都是0.0！

## ✅ 修复方案

### 1. 统一字段名称
修改了v6评分器返回结果，同时包含两个字段名以确保兼容性：
```python
result = {
    'dimension_scores': dim_scores,  # 统一使用这个
    'dim_scores': dim_scores,        # 兼容旧代码
    ...
}
```

### 2. 完善返回字段
添加了缺失的字段：
```python
'grade': self._get_grade_v4(final_score),  # 添加评级（S/A/B/C）
'description': self._generate_description_v6(...)  # 添加推荐理由
```

### 3. 新增推荐理由生成
实现了`_generate_description_v6`方法，自动生成推荐理由：
```python
def _generate_description_v6(self, dim_scores: Dict, final_score: float) -> str:
    reasons = []
    if dim_scores['板块热度'] >= 9:
        reasons.append('板块热度高')
    if dim_scores['资金流向'] >= 10:
        reasons.append('资金持续流入')
    if dim_scores['技术突破'] >= 10:
        reasons.append('技术形态突破')
    # ... 更多条件
```

## 📝 技术总结

这是一个典型的**字段名不一致**导致的bug：
- 评分器使用`dim_scores`
- 主系统提取`dimension_scores`
- 结果：虽然计算正常，但显示时取不到值

**教训**：
1. 不同模块间的数据交换，字段名必须统一
2. 使用`get(key, default)`时要确保key名称正确
3. 返回字典中同时保留两个字段名可以提高兼容性
