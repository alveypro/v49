## 🎯 优化目标
解决用户反馈的问题：**选出的股票有些第二天涨停，有些却跌很多，选股不够稳定。**

---

## 🔍 发现的核心问题

### 1. ❌ 致命缺陷：放量下跌只扣-10分
**问题：** 放量下跌（主力出货）只扣-10分，其他项得分高仍能≥65分入选  
**危害：** 主力出货的股票被选中，第二天大概率下跌  
**原因分析：** 这是导致选出股票有些大跌的主要原因

### 2. ⚠️ 风险盲区：缺少基础风控
**问题：** 没有过滤ST股、跌停股、高位风险股  
**危害：** 高风险股票混入结果，增加亏损概率  
**原因分析：** ST股、跌停股波动极大，不适合普通投资者

### 3. 📉 阈值偏低：65分门槛不够严格
**问题：** 65分让中等质量股票混入  
**危害：** 选股质量参差不齐，稳定性差  
**原因分析：** 门槛低导致"凑分"股票被选中

### 4. 🔄 趋势盲区：只看3天短期数据
**问题：** 只看3天涨跌，忽略中期趋势  
**危害：** 可能选出短期反弹但长期下跌的股票  
**原因分析：** 缺少趋势确认，选到"死猫跳"

### 5. 📍 底部判断不够严格
**问题：** 可能选入刚创新低的股票  
**危害：** 这类股票可能继续下跌  
**原因分析：** 底部判断只看价格位置，未考虑新低风险

---

## 💡 核心优化方案（五重风控）

### 🛡️ 一票否决1：放量下跌直接排除
```python
# ⚠️【一票否决1】放量下跌=主力出货，直接排除！
if vol_ratio > 1.5 and price_chg_3d < -0.02:
    continue  # 直接跳过此股票
```
**效果：** 彻底避免主力出货的股票，大幅减少大跌风险

### 🛡️ 一票否决2：ST/*股直接排除
```python
# ⚠️【一票否决2】ST股直接排除
stock_name = stock_data['name'].iloc[0]
if 'ST' in stock_name or '*' in stock_name:
    continue
```
**效果：** 排除退市风险股，保护本金安全

### 🛡️ 一票否决3：近5日跌停股排除
```python
# ⚠️【一票否决3】近5天有跌停，排除（风险过大）
has_limit_down = any(p < -9.5 for p in pct_chg[-5:])
if has_limit_down:
    continue
```
**效果：** 避免高波动风险股，提高稳定性

### 🛡️ 一票否决4：高位回落风险排除
```python
# ⚠️【一票否决4】高位下跌风险（60日内涨幅>50%且近期下跌）
if len(close) >= 60:
    gain_60d = (close[-1] - close[-60]) / close[-60]
    if gain_60d > 0.5 and price_chg_3d < -0.03:
        continue  # 高位回落，风险大
```
**效果：** 避免高位套牢，减少回撤风险

### 🛡️ 一票否决5：短期均线全向下排除
```python
# 🔥【新增】中期趋势确认：MA5和MA10必须向上
ma5_prev = np.mean(close[-6:-1])
ma10_prev = np.mean(close[-11:-1])
ma5_trend_up = ma5 > ma5_prev
ma10_trend_up = ma10 > ma10_prev

# ⚠️【一票否决5】如果短期均线都向下，排除（下跌趋势）
if not ma5_trend_up and not ma10_trend_up:
    continue
```
**效果：** 确保趋势向上，避免下跌途中的反弹
