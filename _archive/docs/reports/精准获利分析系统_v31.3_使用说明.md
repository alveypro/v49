# 精准获利分析系统 v31.3 稳定版 - 使用说明

## 🎯 系统特点

### 1. **永久独立数据库**
- **数据库路径**: `/Users/mac/QLIB/permanent_stock_database.db`
- **特性**: 独立存储，不会因系统更新而丢失数据
- **容量**: 支持全市场5000+只股票的完整历史数据
- **结构**: 6个核心表，包含索引优化，查询性能优异

### 2. **已修复的问题**
✅ 修复了 `st.set_page_config()` 多次调用错误
✅ 增加了Tushare API超时时间到60秒
✅ 添加了数据更新的3次重试机制
✅ 优化了API调用频率，避免触发限流
✅ 增强了错误处理和日志记录

### 3. **核心功能**
- 📊 全市场股票基础信息管理（4983只股票已成功导入）
- 📈 批量历史交易数据更新（支持自定义日期范围）
- 🔧 自动计算20+项技术指标
- 🎯 多维度精准分析（5个评分维度）
- 💹 智能交易指导（买卖点、止损、持仓建议）
- 🗄️ SQL查询工具（7个预设查询 + 自定义查询）
- 📋 完整的数据更新日志

## 🚀 快速开始指南

### 第一步：启动系统
系统已经在端口 8505 上运行：
```
访问地址: http://localhost:8505
```

### 第二步：更新股票基础数据（如果还未更新）
1. 点击 "SQL数据管理" 标签页
2. 在 "全市场数据更新" 部分
3. 点击 "🔄 更新全市场股票信息"
4. 等待更新完成（约1-2分钟，更新4983只股票）

**状态**: ✅ 已完成（4983只股票已导入）

### 第三步：更新历史交易数据
1. 在 "SQL数据管理" 标签页
2. 选择日期范围（建议从30天前开始）
3. 选择更新股票数量（建议先选100只测试）
4. 点击 "🔄 批量更新交易数据"
5. 系统会自动：
   - 获取日线交易数据
   - 计算技术指标（MA、MACD、RSI、KDJ、BOLL等）
   - 保存到永久数据库

**建议策略**:
- **首次使用**: 先更新100只股票的30天数据（约10-15分钟）
- **日常维护**: 每天更新全市场当天数据（约30-40分钟）
- **深度分析**: 更新1000只股票的90天数据（约2-3小时）

### 第四步：运行精准分析
1. 切换到 "精准分析" 标签页
2. 设置分析参数：
   - 分析日期：当前日期
   - 最低评分阈值：70分（推荐）
   - 分析股票数量：2000只（根据已有数据选择）
3. 点击 "🚀 开始精准分析"
4. 查看分析结果：
   - 总分排名
   - 推荐级别
   - 买卖价格建议
   - 预期收益
   - 风险等级

## 📊 数据库表结构

### 1. stock_basic（股票基础信息）
- 5000+只全市场股票
- 行业、市场、上市日期等基础信息
- 活跃状态标记

### 2. daily_trading_data（日线交易数据）
- 开盘、最高、最低、收盘价
- 成交量、成交额
- 涨跌幅、换手率等

### 3. technical_indicators（技术指标）
- 均线系统（MA5/10/20/60）
- MACD指标
- RSI指标
- KDJ指标
- 布林带
- ATR、OBV等

### 4. analysis_results（分析结果）
- 多维度评分
- 交易建议
- 风险评估
- 历史分析记录

### 5. data_update_log（更新日志）
- 更新类型
- 日期范围
- 成功/失败统计
- 错误信息

## 🔍 SQL查询示例

### 查看已更新的股票数量
```sql
SELECT COUNT(*) as 已更新股票数 
FROM stock_basic 
WHERE is_active = 1
```

### 查看交易数据覆盖情况
```sql
SELECT 
    COUNT(DISTINCT ts_code) as 有交易数据的股票数,
    COUNT(*) as 交易数据总条数,
    MIN(trade_date) as 最早日期,
    MAX(trade_date) as 最新日期
FROM daily_trading_data
```

### 查看技术指标计算情况
```sql
SELECT 
    COUNT(DISTINCT ts_code) as 已计算技术指标的股票数,
    COUNT(*) as 技术指标总条数
FROM technical_indicators
```

### 查找高分股票
```sql
SELECT 
    ts_code, stock_name, total_score, 
    recommendation, analysis_date
FROM analysis_results 
WHERE total_score >= 80
ORDER BY total_score DESC 
LIMIT 20
```
