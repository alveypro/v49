# 🔥 紧急修复：v5.0和v6.0调用错误

**发现时间**: 2024-12-24  
**严重程度**: ⚠️ 高（导致v6.0全部0分）

---

## 🐛 问题描述

用户报告：**v6.0策略扫描2778只股票，但找不到≥60分的股票**

### 根本原因

发现了两个严重的调用错误：

### 1. **v6.0缺少ts_code参数** ⚠️⚠️⚠️

**错误代码**:
```python
# ❌ 错误：只传了stock_data
score_result = vp_analyzer.evaluator_v6.evaluate_stock_v6(stock_data)
```

**v6.0评分器定义**:
```python
def evaluate_stock_v6(self, stock_data: pd.DataFrame, ts_code: str) -> Dict:
```

**影响**:
- v6.0需要`ts_code`来获取外部数据：
  - **板块热度**（25分） ← 完全缺失
  - **资金流向**（20分） ← 完全缺失
  - **相对强度**（10分） ← 完全缺失
- **合计：55分直接丢失！**
- 剩下最多45分，无法达到60分阈值

### 2. **v5.0方法名错误** ⚠️

**错误代码**:
```python
# ❌ 错误：方法名不存在
score_result = vp_analyzer.evaluator_v5.evaluate_stock_v5(stock_data)
```

**正确方法名**:
```python
# ✅ 正确：v5.0的方法名仍然是evaluate_stock_v4
score_result = vp_analyzer.evaluator_v5.evaluate_stock_v4(stock_data)
```

**影响**:
- v5.0扫描时会抛出异常
- 导致v5.0完全无法使用

---

## ✅ 修复方案

### v6.0修复

```python
# 修复前
score_result = vp_analyzer.evaluator_v6.evaluate_stock_v6(stock_data)

# 修复后
score_result = vp_analyzer.evaluator_v6.evaluate_stock_v6(stock_data, ts_code)  # ← 传入ts_code
```

### v5.0修复

```python
# 修复前
score_result = vp_analyzer.evaluator_v5.evaluate_stock_v5(stock_data)

# 修复后
score_result = vp_analyzer.evaluator_v5.evaluate_stock_v4(stock_data)  # ← 使用正确的方法名
```

## 🎯 v6.0数据依赖说明

v6.0是**超短线策略**，依赖Tushare Pro的以下接口：

### 1. 板块热度（25分）

**数据来源**:
```python
- stock_basic()          # 获取行业分类
- concept_detail()       # 获取概念题材
- daily()                # 计算板块涨幅（抽样20只股票）
```

**评分逻辑**:
- 板块3天涨幅>5%：12分
- 板块排名前5：8分
- 有热门概念：5分

### 2. 资金流向（20分）

**数据来源**:
```python
- moneyflow()            # 个股资金流向（需2000积分）
```

**评分逻辑**:
- 连续3天大单流入：10分
- 超大单净流入>3000万：5分
- 北向资金连续买入：5分

### 3. 相对强度（10分）

**数据来源**:
```python
- index_daily()          # 大盘涨跌幅
```

**评分逻辑**:
- 跑赢大盘：0-10分

## ⚠️ Tushare积分要求

| 接口 | 需要积分 | 说明 |
|------|---------|------|
| `stock_basic` | 0 | 免费 ✓ |
| `daily` | 0 | 免费 ✓ |
| `concept_detail` | 200 | 需要积分 |
| `moneyflow` | 2000 | 需要高级积分 ⚠️ |
| `moneyflow_hsgt` | 5000 | 需要VIP积分 ⚠️ |

**如果用户积分不足**:
- 这些接口会返回空数据
- v6_data_provider会返回默认值（0分）
- 但至少不会报错，能获取到基础技术指标分数
