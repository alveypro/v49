## 📊 问题现象

## 🔍 诊断过程

### 第一步：代码错误修复

发现v5评分器存在以下代码错误：

#### 错误1：缺少评分方法
```python
# 错误：调用不存在的方法
score_macd = self._score_macd_trend(indicators)  # ❌ 方法不存在
score_ma = self._score_ma_trend(indicators)      # ❌ 方法不存在
```

**修复方案：**
```python
# 修复：添加v5专属评分方法，调用父类v4方法并按比例转换
def _score_macd_trend_v5(self, ind: Dict, close) -> float:
    """MACD趋势（7分）- 从v4的15分按比例转换"""
    score_v4 = self._score_macd_trend_v4(ind, close)
    return min(score_v4 * (7.0 / 15.0), 7.0)

def _score_ma_trend_v5(self, ind: Dict) -> float:
    """均线多头（7分）- 从v4的10分按比例转换"""
    score_v4 = self._score_ma_trend_v4(ind)
    return min(score_v4 * (7.0 / 10.0), 7.0)
```

#### 错误2：止损止盈字段错误
```python
# 错误：访问不存在的字段
'stop_loss': self._recommend_stop_loss_v4(close, indicators)['price']  # ❌ 没有'price'字段
```

**修复方案：**
```python
# 修复：使用正确的字段名
'stop_loss': self._recommend_stop_loss_v4(close, indicators)['stop_loss']
'take_profit': self._recommend_stop_loss_v4(close, indicators)['take_profit']
```

## 🎯 问题根源分析

### 核心问题：v5.0评分标准过于严格

v5.0"启动确认版"的设计理念是**在股票已经确认启动后买入**，要求：

1. **启动确认（20分）** - 要求股票已经明确启动：
   - 站上20日线和60日线（8分）
   - 连续阳线（5天4阳，6分）
   - 放量上涨（涨幅>5%且量比>1.5，6分）
   
2. **主力行为（18分）** - 要求明显的资金流入：
   - 量比>2.0（9分）
   - 筹码高度集中（9分）
   
3. **涨停基因（8分）** - 要求历史爆发力：
   - 20天内有涨停（4-8分）
   - 多次大阳线（2分）

### 为什么评分普遍偏低？

#### 1. 启动确认得分低（平均7.0/20分）

**原因：**
- 要求同时满足"站上双均线 + 连续阳线 + 放量上涨"才能得高分
- 当前市场环境下，大部分股票处于震荡或下跌状态
- 真正"已经启动"的股票很少

**数据支持：**
- 500只股票中，启动确认得分≥15分的只有约10%
- 大部分股票得分在0-10分之间

#### 2. 涨停基因得分极低（平均0.9/8分）

**原因：**
- 要求20天内有涨停才能得分
- 涨停是极端行情，大部分股票很少涨停
- 当前市场环境下，涨停股票更少

**数据支持：**
- 500只股票中，约90%的股票涨停基因得分为0
- 只有极少数股票有涨停记录

#### 3. 主力行为得分偏低（平均7.2/18分）

**原因：**
- 要求量比>2.0才能得高分（9分）
- 量比>2.0意味着成交量是平时的2倍以上，属于明显放量
- 大部分股票量比在1.0-1.5之间

## 💡 优化建议

### 方案A：降低评分阈值（简单快速）

**建议：**
- 将默认阈值从60分降低到**50分**
- 强势启动模式：从70分降低到**60分**
- 即将爆发模式：从65分降低到**55分**
- 潜在机会模式：从60分降低到**50分**

**效果预测：**
- ≥50分：62只（12.4%）- 可以选出一定数量的股票
- ≥55分：38只（7.6%）
- ≥60分：23只（4.6%）

**优点：**
- 修改简单，只需调整界面默认值
- 立即见效

**缺点：**
- 降低了选股标准，可能选出质量不够高的股票

---

### 方案B：优化评分权重（推荐）⭐

**核心思路：降低"启动确认"和"涨停基因"的权重要求**

#### 调整1：启动确认评分标准放宽

**当前标准（过严）：**
```python
if current_price > ma20 and current_price > ma60:
    score += 8  # 必须站上双均线
elif current_price > ma20:
    score += 5  # 只站上20日线
```

**优化建议：**
```python
if current_price > ma20 and current_price > ma60:
    score += 8  # 站上双均线
elif current_price > ma20:
    score += 6  # 站上20日线（提高到6分）
elif current_price > ma60:
    score += 4  # 站上60日线（提高到4分）
else:
    score += 2  # 接近均线也给分（新增）
```

#### 调整2：放量上涨标准放宽

**当前标准（过严）：**
```python
if price_chg_5d > 5 and vol_ratio > 1.5:
    score += 6  # 涨幅>5%且量比>1.5
elif price_chg_5d > 3 and vol_ratio > 1.2:
    score += 4
```

**优化建议：**
```python
if price_chg_5d > 3 and vol_ratio > 1.3:  # 降低要求
    score += 6
elif price_chg_5d > 1 and vol_ratio > 1.1:  # 降低要求
    score += 4
elif price_chg_5d > 0 and vol_ratio > 1.0:
    score += 2
```

#### 调整3：涨停基因不作为必要条件

**当前问题：**
- 涨停基因8分，但大部分股票得0分
- 导致总分偏低

**优化建议：**
- 保持涨停基因评分逻辑不变
- 但在协同加分中，不强制要求涨停基因
- 有涨停基因是加分项，没有也不影响基础分
