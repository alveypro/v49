# 🔧 修复v6/v7回测不显示问题

## 📊 问题诊断结果

### ✅ 已确认的问题

**根本原因**：数据列名不匹配导致v6和v7评分失败

| 策略 | 期望列名 | 实际列名 | 结果 |
|------|---------|----------|------|
| v4.0 | `close` | `close` | ✅ 正常 |
| v5.0 | `close` | `close` | ✅ 正常 |
| v6.0 | `close_price` | `close` | ❌ 失败 |
| v7.0 | `close_price` | `close` | ❌ 失败 |

### 📍 问题位置

**文件**: `终极量价暴涨系统_v49.0_长期稳健版.py`  
**行号**: 6631-6636  
**代码**:

```python
# 重命名列以匹配评分器期望
df = df.rename(columns={
    'close_price': 'close',    # ❌ v6/v7需要close_price
    'open_price': 'open',
    'high_price': 'high',
    'low_price': 'low'
})
```

---

## 🔧 解决方案

### 方案1: 修改列名重命名逻辑（推荐）⭐⭐⭐⭐⭐

**优势**: 一次修复，永久解决  
**难度**: 简单  
**风险**: 低

#### 修改步骤

找到 `终极量价暴涨系统_v49.0_长期稳健版.py` 第6631行，**删除列名重命名代码**：

```python
# ❌ 删除这段代码
# df = df.rename(columns={
#     'close_price': 'close',
#     'open_price': 'open',
#     'high_price': 'high',
#     'low_price': 'low'
# })
```

#### 原因

- v4和v5的评分器已经适配了多种列名格式
- v6和v7必须使用`close_price`格式
- 数据库返回的就是`close_price`，不需要重命名

---

### 方案2: 针对性处理（临时方案）⭐⭐⭐

修改v6和v7的回测方法，在评分前恢复列名：

```python
# 在调用v6评分前
if 'close' in df.columns and 'close_price' not in df.columns:
    df = df.rename(columns={
        'close': 'close_price',
        'open': 'open_price',
        'high': 'high_price',
        'low': 'low_price'
    })
```

## 💡 为什么v4/v5能用

v4和v5的评分器在设计时考虑了列名兼容性：

```python
# v4评分器内部会自动处理
if 'close' in data.columns:
    close_col = 'close'
elif 'close_price' in data.columns:
    close_col = 'close_price'
```

而v6和v7是后期开发的，直接hard code了`close_price`。
