## 📧 方式1: 邮件通知（推荐）

### 优点
- ✅ 最简单、最稳定
- ✅ 支持所有邮箱
- ✅ 可发送详细内容
- ✅ 无需第三方服务

### 配置步骤

#### 1. 获取邮箱授权码

##### QQ邮箱（推荐）

1. 登录 [QQ邮箱](https://mail.qq.com)
2. 点击「设置」→「账户」
3. 找到「POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV服务」
4. 开启「POP3/SMTP服务」或「IMAP/SMTP服务」
5. 点击「生成授权码」
6. **保存授权码**（16位字符串）

##### 163邮箱

1. 登录 [163邮箱](https://mail.163.com)
2. 点击「设置」→「POP3/SMTP/IMAP」
3. 开启「IMAP/SMTP服务」
4. 点击「授权密码管理」→「新增授权密码」
5. **保存授权码**

##### Gmail

1. 登录 [Google账户](https://myaccount.google.com)
2. 进入「安全性」
3. 开启「两步验证」
4. 生成「应用专用密码」
5. **保存密码**

#### 2. 配置文件示例

```json
{
  "enabled": true,
  "email": {
    "enabled": true,
    "smtp_server": "smtp.qq.com",
    "smtp_port": 587,
    "sender_email": "your_email@qq.com",
    "sender_password": "your_16_char_authorization_code",
    "receiver_emails": [
      "receiver@example.com"
    ]
  }
}
```

#### 3. 常见邮箱SMTP配置

| 邮箱 | SMTP服务器 | 端口 |
|------|-----------|------|
| QQ邮箱 | smtp.qq.com | 587 或 465 |
| 163邮箱 | smtp.163.com | 25 或 465 |
| 126邮箱 | smtp.126.com | 25 或 465 |
| Gmail | smtp.gmail.com | 587 或 465 |
| Outlook | smtp.office365.com | 587 |

#### 4. 测试邮件

```bash
cd /Users/mac/QLIB
python -c "from notification_service import NotificationService; NotificationService().test_notification()"
```

## 💬 方式2: Server酱（微信推送）⭐⭐⭐⭐⭐

### 优点
- ✅ 推送到**个人微信**
- ✅ 配置超简单（只需1个key）
- ✅ 完全免费
- ✅ 秒级到达

### 配置步骤

#### 1. 注册Server酱

1. 访问 [https://sct.ftqq.com](https://sct.ftqq.com)
2. 使用GitHub账号登录（没有就注册一个）
3. 扫码绑定微信
4. 复制您的 **SendKey**（形如：`SCT123456TxxxxxxxxxxxxxxxxxxxZ`）

#### 2. 配置文件

```json
{
  "enabled": true,
  "serverchan": {
    "enabled": true,
    "sendkey": "SCT123456TxxxxxxxxxxxxxxxxxxxZ"
  }
}
```

#### 3. 测试推送

```bash
python -c "from notification_service import NotificationService; ns = NotificationService(); ns.send_serverchan('测试', '如果您在微信收到这条消息，说明配置成功！')"
```

**优点**：
- 推送到个人微信，不错过任何消息
- 免费、稳定、快速

## 📱 方式3: Bark（iOS推送）

### 优点
- ✅ iOS原生推送
- ✅ 配置超简单
- ✅ 完全免费
- ✅ 秒级到达

### 配置步骤

#### 1. 下载Bark App

- App Store搜索「Bark」
- 或访问：[https://apps.apple.com/cn/app/bark/id1403753865](https://apps.apple.com/cn/app/bark/id1403753865)

#### 2. 获取Device Key

1. 打开Bark App
2. 复制您的推送地址（类似：`https://api.day.app/YOUR_DEVICE_KEY/`）
3. 提取 `YOUR_DEVICE_KEY` 部分

#### 3. 配置文件

```json
{
  "enabled": true,
  "bark": {
    "enabled": true,
    "device_key": "YOUR_DEVICE_KEY"
  }
}
```

#### 4. 测试推送

```bash
python -c "from notification_service import NotificationService; ns = NotificationService(); ns.send_bark('测试', '如果您的iPhone收到这条推送，说明配置成功！')"
```

## ❓ 常见问题

### Q1: 邮件发送失败，显示"535认证失败"

**A**: 可能原因：
1. 没有使用授权码，而是用了登录密码
2. 授权码输入错误
3. 没有开启SMTP服务

**解决方法**：
1. 重新获取授权码
2. 确认授权码正确无误
3. 检查邮箱设置中SMTP服务已开启

---

### Q2: Server酱推送不到微信

**A**: 可能原因：
1. SendKey输入错误
2. 没有绑定微信
3. 微信解绑了

**解决方法**：
1. 访问 [https://sct.ftqq.com](https://sct.ftqq.com)
2. 检查SendKey是否正确
3. 重新绑定微信

---

### Q3: 收不到推送

**A**: 检查清单：
1. ✅ 配置文件中 `"enabled": true`
2. ✅ 对应通知方式的 `"enabled": true`
3. ✅ 配置信息正确填写
4. ✅ 运行了 `scheduler_service.py`
5. ✅ 测试通知能正常发送

---

### Q4: 定时任务没有运行

**A**: 检查：
1. 是否运行了 `scheduler_service.py`
2. 查看日志：`tail -f scheduler.log`
3. 手动测试：`python demo_trading_assistant.py`
