# T12 Governance Summary Invariants

1. 只读
- T12 Governance Summary 只能展示摘要，不提供按钮、表单、bridge、动作挂钩或治理操作。

2. 不回写
- 治理摘要字段不得写回统一事实层，也不得写入新的治理事实存储。

3. 不入 `/stock`
- 治理摘要区只允许挂载在 `/T12` 只读作用域，不进入 `/stock` 页面。

4. 不新增事实源
- 治理摘要只能消费共享制度事实层白名单字段，不新增 JSON、DB、缓存或中间态事实源。

5. 不扩成交互控制台
- `view=t12` 只能表示只读镜像作用域，不得演化为控制台、操作台或治理入口。

6. 缺失值只允许三种
- 缺失值输出只允许：
  - `暂缺`
  - `待补充`
  - `降级说明`

7. `risk_level` 不能独立构成制度阻断
- `risk_level=high/critical` 只有在已有既有阻断事实支持时，才能参与“不可推进”摘要。
- 不允许把 `risk_level` 单独升级成新的制度判断规则。

8. 模板不承担治理判断
- 推进状态、主要阻断、治理备注的判断必须在 Python 展示层完成。
- 模板只能消费已经生成好的 ViewModel。
