# STOCK PRIMARY RESULT REFINEMENT SPEC

## Purpose

本文件定义 `/stock` canonical 单轨运行后的结果精修规则。

目标：

- 保持结论层更短、更稳、更像主结果中心
- 保持解释层排序稳定、信息压缩
- 保持边界层轻量，不演变成 `/T12` 替身

## Conclusion Rules

- 结论层优先承载：对象、主结果标签、风险提示
- 状态标签只允许使用既有受控集合
- 结论层文案必须短句化，不扩写成长解释
- 结论层不得混入治理摘要、主站平台叙事、产品矩阵语言

## Explanation Rules

- 解释层固定为少量压缩信息，不扩成正文区
- 排序规则：
  - 历史摘要优先
  - 来源引用固定放最后
- 解释项数量和长度必须受控
- 解释层只能辅助，不得压过主结论

## Boundary Rules

- 边界层固定为轻量三句
- 必须继续提示 `/T12` 是治理主解释权位置
- 不得展开推进状态、阻断、治理备注等治理内容

## Allowed Enhancements

- 文案收敛
- 标签标准化
- 解释层压缩
- 观测字段稳定化

## Disallowed Changes

以下改动会破坏主结果中心感：

- 解释层变长并压过结论层
- 边界层膨胀成治理摘要区
- 结论层混入治理语言或主站品牌语言
- 新增与主结果无关的展示层级
