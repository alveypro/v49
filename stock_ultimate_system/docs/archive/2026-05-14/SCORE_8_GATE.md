# SCORE 8 Gate

目的：把“8/10”从主观感觉改成可审计的验收标准。  
适用对象：每周复盘、阶段评审、对外汇报、内部优先级排序。  
评分原则：`8/10` 代表行业内已经能被严肃对待，但还不是顶级垄断级；必须有连续证据，不接受“结构存在但运行不足”。

## 1. 研究流程工程化

### 8分定义
- 研究不是零散试验，而是标准化流水线。
- 输入明确，过程可复现，输出有固定工件。
- 失败有归因，晋升有门槛。

### 当前差距
- lifecycle、gate、preflight、ledger、artifact 体系已成型。
- 差距在连续运行证明还不够厚，部分链条仍带过渡态。

### 必须看到的证据
- `candidate -> lifecycle -> observation -> terminal -> ledger` 连续运行记录。
- 每日工件稳定生成。
- 失败样本进入 attribution。
- 同一研究流程能被重复执行，不靠个别人记忆。

### 不达标时的真实扣分原因
- 流程只写在文档里，没有连续运行样本。
- 工件缺失时靠人工补。
- 同一对象在不同阶段口径漂移。
- 研究结果无法复现。

## 2. 治理与门禁

### 8分定义
- 门禁不只是显示状态，而是真正能阻断错误、允许正确流转。
- 每次裁决可追溯、有主因、有下一步动作。

### 当前差距
- 这是当前最接近 8 的维度。
- 差距主要在边缘条件、namespace/scope 漂移治理，以及残留文案和入口信号统一。

### 必须看到的证据
- promotion gate、preflight、handoff gate 长期生效。
- `blocked / pass / degraded / pending_window / failed` 语义统一。
- 每个门禁都有主因和 `next action`。
- `release / baseline / rollback` 有真实决策历史。

### 不达标时的真实扣分原因
- 门禁只是展示，不改变行为。
- 页面说可推进，底层工件链实际不允许。
- 例外放行没有审计记录。
- 边界解释权分散。

## 3. 自动化与产物链路

### 8分定义
- 系统每天自动产出关键工件。
- 产物链路稳定、可追踪、可回放，不靠手工拼装。

### 当前差距
- artifact 数量已经很多。
- 差距在连续时间序列，而不是 latest 文件数量。

### 必须看到的证据
- 每日自动生成关键 artifacts。
- artifact 带时间戳、来源任务、上游状态。
- 失败和降级也有产物，不静默丢失。
- 周期内可回放任一天状态。

### 不达标时的真实扣分原因
- 只有 latest，没有 history。
- 工件存在但无法追溯生成链。
- 失败时直接断档。
- 产物靠人工拼接。

## 4. 可观测性与运维严谨性

### 8分定义
- 系统能连续观察健康度、异常、降级和恢复。
- 有运行纪律，不是脚本存在就算通过。

### 当前差距
- 已有 health 脚本和 operations artifacts。
- 差距在连续运行视角还不够厚。

### 必须看到的证据
- 7-30 天连续健康轨迹。
- 运行日报：`success / blocked / failed / degraded`。
- 红灯不过夜机制。
- 故障类型、修复动作、恢复结果有记录。

### 不达标时的真实扣分原因
- 只看一次健康检查。
- degraded 没有单独处理。
- 出问题只能临时排查，不能快速定位。
- 没有运维纪律，只有脚本。

## 5. 策略证据强度

### 8分定义
- 不是“有过几次好结果”，而是样本足够、口径稳定、回撤可解释。
- 超额不能是单环境幻觉。
- `primary_result` 和 `candidate_basket` 两条流都成立。

### 当前差距
- 这是当前最明显短板。
- 体系已经承认自己还在 accumulating，这种口径是正确的，但也意味着尚未到 8。

### 必须看到的证据
- `primary_result` 连续 ledger。
- `candidate_basket` 连续 ledger。
- `20 / 60 / 120` 交易日地板推进。
- `success_rate / drawdown / excess_return / failure_distribution` 固定统计。
- 不同市场状态下有分环境样本。

### 不达标时的真实扣分原因
- 样本太薄。
- 只展示单次漂亮结果。
- basket 流长期空。
- 混样本、改口径、事后解释过强。

## 6. 执行闭环能力

### 8分定义
- 系统能从候选走到观察、终局、记账、复盘。
- 这条链长期不断，不依赖人工对齐。

### 当前差距
- 闭环设计已经成型。
- 8 分要求的是连续完成闭环，而不是只知道闭环图该怎么画。

### 必须看到的证据
- 每天有 terminal 状态。
- ledger 每天新增。
- failed terminal 自动进入 `review / feedback`。
- blocked 有唯一主因。
- current candidate 与 lifecycle pointer 长期一致。

### 不达标时的真实扣分原因
- `observation / terminal / ledger` 任何一环经常断。
- blocked 原因模糊。
- failure 不进入学习链。
- current candidate 和 lifecycle pointer 常错位。

## 7. 实盘可放大性

### 8分定义
- 不是“看起来能放大”，而是容量、流动性、集中度、执行压力都经过约束与验证。
- 放大讨论必须有实证基础。

### 当前差距
- 当前不能乐观打高分。
- 这项天然最慢，受资金规模、交易摩擦、市场环境影响，不是工程堆出来的。

### 必须看到的证据
- capacity stress 报告。
- liquidity coverage。
- concentration constraints。
- 不同规模下的 stress score。
- “研究可用”与“放大量可用”明确分开。
- 放大后策略行为仍稳定。

### 不达标时的真实扣分原因
- 把研究结果直接外推成资金容量。
- 没有滑点或流动性证据。
- 没有分规模验证。
- 风险分数和实际执行压力脱节。

## 8. 产品边界清晰度

### 8分定义
- 用户、团队、投资人都能说清：
- 主站做什么。
- `/stock` 做什么。
- `/T12` 做什么。
- `/apex` 只是 namespace，不是第四 system。

### 当前差距
- 这一项已接近 8。
- 差距主要是残留文案、部署一致性和外部表达一致性。

### 必须看到的证据
- 主规范、registry、UI、测试一致。
- `/apex` 文案统一为 namespace 口径。
- 首页不再抢 `/stock` 和 `/T12` 的职责。
- 外部说明材料口径一致。

### 不达标时的真实扣分原因
- 入口职责混淆。
- `/apex` 被当第四系统表达。
- 主站替代结果页或治理页。
- 文档、代码、页面三套说法。

## 9. 生产可靠性

### 8分定义
- 主干稳定、回归稳定、部署稳定、回滚稳定。
- 不是偶尔可用，而是长期可依赖。

### 当前差距
- 主干已经打到全绿过，这是必要前提。
- 8 分要求连续稳定，不是一次绿灯。

### 必须看到的证据
- 连续多轮全量回归全绿。
- 关键入口探活稳定。
- 未知路径、边界条件、降级状态有受控处理。
- `baseline / release / rollback` 真正可用。
- 部署后探活和本地契约一致。

### 不达标时的真实扣分原因
- 主干偶发红灯。
- 线上入口映射和本地契约不一致。
- 边界输入直接打崩页面。
- 没有回滚纪律。

## 10. 行业竞争力综合评价

### 8分定义
- 严肃投资人、严肃客户、严肃合作方会承认：这不是一个讲故事的工具，而是一套开始形成复利的系统。
- 复利来自数据积累、工件积累、失败学习、组织纪律、产品边界和可复制流程。

### 当前差距
- 当前最缺的不是野心，而是能让外部人一眼信服的连续证据。
- 工程骨架已成，竞争力证明还在追赶。

### 必须看到的证据
- 3 个月以上稳定运行轨迹。
- 3 个月以上持续证据积累。
- 能讲清楚商业价值和不可替代性。
- 有可复制的组织节奏，不依赖单点英雄。
- 对外口径不过度承诺。

### 不达标时的真实扣分原因
- alpha 证据薄。
- 组织流程强但商业逻辑弱。
- 演示像系统，复利还没形成。
- 外部人看不出壁垒来自哪里。

## 当前真实位置

### 已接近 8 的项
- 研究流程工程化
- 治理与门禁
- 产品边界清晰度

### 有机会在 30-60 天打到 8 的项
- 自动化与产物链路
- 可观测性与运维严谨性
- 执行闭环能力
- 生产可靠性

### 不能靠意志硬拉到 8 的项
- 策略证据强度
- 实盘可放大性
- 行业竞争力综合评价

这三项必须靠时间、样本和稳定执行去换。

## 使用规则

每周复盘只做两件事：

1. 给九项分别打分。  
   每个分数后面必须跟一句证据，不准空打。

2. 对低于 8 的项只回答四个问题：
- 差在哪里
- 差的是结构、运行还是样本
- 本周补了什么证据
- 为什么还不能给 8

## 禁止事项

- 禁止把页面复杂度当成熟度。
- 禁止把制度文本当证据样本。
- 禁止用单次结果替代连续证明。
- 禁止在证据不足时对外做过强承诺。
- 禁止把“有潜力”说成“已经领先”。
