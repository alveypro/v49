# /stock 第二阶段 UI 重构清单

## 目标

本清单用于把 `airivo.online/stock` 从“已去工程污染的正式研究终端”推进到“接近行业头部产品表达”的第二阶段重构。

这一阶段不再解决事实链真实性，不再解决 formal-vs-apex 边界，也不再以补更多状态映射为主。

这一阶段只解决一件事：

`让 /stock 的正式主链信息，以更清晰、更稳、更快读的产品结构对外表达。`

## 一、必须保留的正式主链信息

首屏必须保留，而且必须让用户在不展开任何折叠区、不阅读内部术语的前提下就能读到：

1. 当前状态
当前处于：
- 样本积累中
- 可继续观察
- 晋级锁定
- 待复核

状态必须是正式研究终端口径，不能退回原始工程枚举。

2. 当前对象
当候选未形成时，首页只能明确展示：
- 当前 formal 主结果对象
- 该对象是否只是“当前研究对象”
- 不能把它误表达成正式候选、正式推荐、Top1 候选结论

3. 当前边界
首页必须持续明确：
- 当前页面是研究参考，不构成收益承诺
- 当前阶段是否允许执行
- 当前是否只是等待样本形成或等待复核

4. 下一步动作
首页必须持续给出唯一下一步，例如：
- 继续观察
- 等待新增样本
- 等待数据补齐
- 进入人工复核

不允许首页同时抛出多个并列“下一步”，增加认知分叉。

5. 最小证据摘要
首页只能保留一层可快读的证据摘要，例如：
- 健康分
- 候选产物时间
- 数据最新交易日
- 样本进度

证据摘要只能回答“判断是否站得住”，不能在首页塞入长段解释文本。

## 二、必须下沉的内部信息

以下信息必须继续存在，但不得与正式首页并列争夺首屏注意力：

1. 内部运维状态
包括但不限于：
- partial_success
- runtime budget
- task elapsed
- profile short
- validation skipped
- grid_backtest
- repair / rollback / governance 过程细粒度状态

这些信息只能进入折叠区、runbook、ops 入口或下载产物。

2. 原始工程原因词
包括但不限于：
- gate blocked
- top_industry
- batch_prediction_timeout
- source path
- raw generation mode
- history source file

首页只能展示其产品化后的解释，不得直接裸出原词。

3. 诊断附录
以下内容必须后移：
- 全A预筛底表
- 漏斗细表
- 长窗验证细表
- 策略诊断细表
- T12 最小镜像细节

它们可以存在，但只能作为“下钻阅读面”，不是首页的主叙事。

4. 内部复核面
运维控制台、研究闭环、专家摘要、辅助图形，必须明确成为：

`正式研究终端的内部复核面`

而不是继续被用户理解成首页主面的一部分。

## 三、首屏卡片与层级重排方案

### A. 首屏结构

首屏只保留四层：

1. 顶栏状态带
只保留：
- 数据最新交易日
- 候选产物时间
- 观察窗口
- 自动链路
- 更新状态

每项必须是正式口径，不得出现工程枚举。

2. 对外决策脊柱
保留：
- 当前状态
- 下一次检查
- 主结果证据进度
- 候选篮证据进度
- 晋级门禁

这是首页第一信息面，不允许被其它卡片稀释。

3. 主结论区
只保留四张主卡：
- 状态
- 证据
- 边界
- 下一步

这四张卡是正式研究终端的核心判断层，必须始终位于首页中轴，不被运维或附录卡片打断。

4. 当前对象区
当候选未形成时：
- 卡片标题必须是“当前对象”或等价表述
- 不允许使用“优先标的”“Top1 候选”等高暗示词
- 视觉权重必须低于主结论区，不得让用户误读为正式推荐位

### B. 必须降级的卡片

以下卡片不能继续维持当前权重：

1. 研究闭环 / 专家摘要
必须保留，但只能是折叠后的二级复核区，不应继续像主内容块一样占大面积。

2. 辅助图形
保持折叠，继续后移。

3. 内部运维
必须继续收紧，目标是：
- 默认不抢首屏
- 文案不再偏工程诊断
- 改写成“内部复核入口”

### C. 当前对象卡的重构要求

当前对象卡应从“放大对象”改为“约束对象理解”：

1. 标题
优先使用：
- 当前对象
- 当前研究对象

避免：
- 优先标的
- Top1
- 冠军候选

2. 主文案
必须显式提醒：
- 当前只是 formal 主结果对象
- 候选链尚未闭合
- 不应提前放大为正式候选结论

3. 元信息
只保留少量标签，例如：
- 当前结果类型
- 研究状态

不再使用空标签、占位标签或无语义短 chips。

### D. 证据卡重构要求

证据卡当前信息太长，必须分层：

1. 首层只保留：
- 健康分
- 候选生成时间
- 样本进度

2. 长文本说明下沉到：
- 展开说明
- 附录
- review runbook

不允许继续把长句塞进窄卡导致断行破碎。

## 执行门槛

本清单完成前，不得宣称：
- `/stock` 已达到行业第一产品面
- 首页产品重构已完成

本清单完成后，才能把当前阶段的 UI 结论从：

`已去工程污染`

推进为：

`已完成正式研究终端第二阶段产品重构`

## 禁止事项

1. 不允许重新把 `apex` 拉回正式并列面。
2. 不允许为追求“信息完整”重新把内部运维信息抬回首页主层。
3. 不允许只修局部文案碎片，而不处理卡片层级与视觉权重。
4. 不允许把“当前对象”视觉上做成正式推荐位。
5. 不允许把更多工程事实链 JSON 直接暴露进正式首页。
